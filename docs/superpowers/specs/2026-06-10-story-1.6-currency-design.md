## Goal

The engine's currency foundation: vendored reference dataset with localized display + reverse
symbol lookup, the ENT-011 resolution API for EP-2's parser, engine-internal USD normalization,
and the 2-level rate cache (local IDB → OER-proxy Cloud Function → Firestore → OER) with the
proxy ported from prior art and hardened per ENT-004.

## Locked decisions

1. **Proxy request shape: DATE-ONLY** (port prior-art `getUSDRates`). Request `{date}`, response =
   the full day's USD table (~233 currencies, a few KB). Engine extracts rates client-side and
   caches the whole table by date. Strictly stronger minimization than ENT-004's old `{currency,
   date}` phrasing (KB already updated by the PM). Whitelist reduces to date validation
   (format `YYYY-MM-DD` + range 1999-01-01‥today).
2. **Dataset: VENDOR the two JSONs** from `@abc-budget/currencies@1.0.50` (GitHub Packages needs a
   token even for public installs — OSS friction). `currencies.json` (233 entries:
   `{code, defaultFractionDigits, numericCode, localizedData.{en,uk}.{symbol,name}, specialSymbols[]}`)
   + `locale2currency.json` (764 `{locale, currency}`) → `packages/engine/src/internal/currency/data/`
   with `PROVENANCE.md` (package@1.0.50, source `D:\abc-budget\currency-data`, vendored date,
   re-vendor procedure). Same diffable-on-update discipline as `altus.css`/fonts.
   Carry-forward: switch to a normal dep IF the package ever lands on tokenless npmjs.
3. **The remote seam = prior-art `ExchangeRateApi`** (`getExchangeRate(base, date)` — base pinned
   `'USD'`). The engine defines/owns the interface; **`apps/web` implements it** with the Firebase
   SDK (app + functions callable + App Check — the only firebase-js code in the repo) and injects it
   at engine-client creation via an internal factory option. Tests mock the interface — no Firebase
   in tests. **`EngineClient` does NOT grow; contract stays 1** (no UI consumer yet; EP-2's surface
   growth is the bump-rules moment).
4. **`functions/` lives OUTSIDE the pnpm workspace** (own package.json; firebase-tools owns its
   install/build — prior-art isolation). *(Founder note #1):* its pure-helper tests are wired into
   the root gate via a **root `test:functions` script** so the DoD gate actually executes them.

## Prior-art reuse map (verified by reading the source)

- **`getUSDRates` function** (`.deprecated/firebase/functions/src/index.ts`) — PORT + adapt exactly:
  (a) ⚠ `request.auth` check → **`enforceAppCheck: true`** (prior art used Firebase Auth — HC-1
  violation in Core); (b) **+ budget cap** — Firestore transactional monthly counter on OER fetches
  (cache hits free; default cap 1,000/mo = OER free tier; `resource-exhausted` beyond);
  (c) whitelist = date validation incl. range (prior art validated format only);
  (d) CORS → `abc-budget-2d379.web.app`/`.firebaseapp.com` + localhost dev/preview.
  KEEP: `defineSecret("OPENEXCHANGERATES_APP_ID")`, europe-west1, Firestore doc-per-date immutable
  cache, the overall flow (Firestore → OER → save → return).
- **`exchange-rate/` module** (`webapp/libs/engine/src/exchange-rate/` — api, cached-api, dao,
  service, types + 4 spec files) — PORT; the dao targets the 1.2-ported `store/idb` as-is.
  `ExchangeRateEntity {base, date, ...rates}` is ALREADY whole-table-per-(base,date) — matches the
  date-only decision natively (base pinned `'USD'`). Same port discipline as 1.2 (diff-audited,
  Container→DbProvider-style adaptations only, specs come along).
- **`currency/` prior art** (cache/fetch-init) — NOT ported: it fetched currencies at runtime; we
  vendor the data statically. The reference module is NEW (small) over the vendored JSONs.

## 2. Rate flow (Q-006: immutable forever)

Engine `CachedExchangeRateApi` (ported): **local IDB (base,date) table → injected remote
`ExchangeRateApi`**. Function: **Firestore doc-per-date → OER historical → save → return**.
Second identical request anywhere in the chain = cache-only. USD-normalization `service.ts`
(ported) converts amounts via the cached tables — engine-internal, never surfaced (HC-2/3).

## 3. Migration v2 (the 1.2 framework's first real store)

`ENGINE_MIGRATIONS` appends step 2 creating `exchangeRates` (prior-art `EXCHANGE_RATES_STORE_CONFIG`:
keyPath `['base','date']`). Append-only ✓. **The 1.2 anchor test pinning `ENGINE_MIGRATIONS.length === 1`
gets a legitimate update to 2** (explicit, not weakening); fresh≡upgrade re-verified at v2.
