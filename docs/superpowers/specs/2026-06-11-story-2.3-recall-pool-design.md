## Goal

Recall rebuilt at columnName granularity over a persisted pool (migration v3, alongside the engine
settings store), the stage orchestration ported INTACT (incl. its 1,347-line suite), and the
row-generator REBUILT (collect-don't-throw + COUNTERPARTY field) — closing the engine pipeline
end-to-end: decode → stage1 → stage2 (recall-prefilled) → typed transaction rows.

## Locked decisions (founder)

1. **Collision = single entry + save-collision signal.** One pool entry per key. Save: identical →
   no-op; differing → collision descriptor **with `kind: 'type-change' | 'params-change'`** (S3b's
   2.8 confirm copy needs the distinction; one field now vs a descriptor migration later); LWW
   after explicit confirm. No history. Accepted trade-off: two banks alternating on a shared name
   = one confirm per flip (revisit only on real-use evidence).
2. **Pool key = `columnName.normalize('NFC').trim()`.** NFC is canonical equivalence only (merges
   byte-different encodings of identical characters — xlsx extractors emit either form); case +
   internal whitespace PRESERVED (latin-i «Дата i час» ≠ cyrillic-і variant — different banks,
   honest). **Refinement (founder): the STORED columnName IS the normalized key — NFC+trim applied
   BEFORE save, not only at lookup.** Pinned test: save with a raw NFD/padded name → exactly ONE
   entry, keyed normalized; lookup by any variant (NFD, padded, exact) hits it.
3. **Stage3 boundary: row-generator IN (rebuilt), categorization OUT (EP-4).** The rebuild =
   collect-don't-throw per FEAT-022 (wedge REBUILD target #2) + the COUNTERPARTY output field;
   2.5 extends the row-generator with pseudo-ops.
4. **Acceptance addition: the real-statement local spec extends END-TO-END** — the three real files
   flow decode→stage1→stage2(recall)→typed transaction rows locally with hard-asserted row counts
   (the 2.1 pattern, one level deeper — checkpoint (b)'s spine early).
5. *(Noted, accepted, no change):* `col_N` pool entries are position-dependent across different
   banks' empty-header files — a recall hit may prefill an unrelated column. GUESSED ◇ + confirm
   covers it by design; revisit on evidence.

## 2. Recall contract

- GUESSED ◇ is a first-class state on prefilled columns (≠ confirmed ✓ ≠ UNKNOWN ?) — VIS-009
  one-click confirm covers ambiguity; auto-detect results are GUESSED too.
- Empty pool (first import) → everything UNKNOWN, manual (baseline, FEAT-013).
- Unknown names STAY UNKNOWN (never auto-guessed unless the auto-detect flag is on — and then
  deterministically + GUESSED).
- N-of-M derived from recall results (no stored counter).
- `use_base` resolution: reads base currency from settings; unset → loud resolution error.
