## Goal

The engine runs in a dedicated Worker behind the same `EngineClient`; the transport is
production-grade (drain, timeouts, structured errors, contract handshake); the client surface
grows the real import API (serializable session protocol); the dead prior-art format machinery
is excised; the lazy-dep build discipline is proven; `./qa` sunsets.

## Locked decisions (founder, 2026-06-12)

1. **Worker host: dedicated Worker per tab.** Multi-tab model on record: "loud, not
   coordinated" (consistent with the 2.4 stale-window acceptance). The onblocked UX is the
   multi-tab answer: version-change/blocked → loud "close other tabs / reload" state, never a
   hang. **Pin:** tab closed mid-job → the worker dies with it and NO orphaned engine state
   survives — IDB transaction atomicity is the guarantee; assert it (kill mid-import → reopen
   → stores contain either none or complete units, never partials).
2. **Contract: exact-match handshake.** `hello {contract: N}` is the FIRST message; the worker
   replies with its own; ANY mismatch → structured `ContractMismatchError` BEFORE any job is
   accepted → loud reload-the-app state. Bump rule: ANY change to method set, signatures, or
   wire-envelope semantics bumps the integer — additions included. (Resolves the 1.1
   contract-versioning carry-forward.) **Integration refinement:** mismatch implies a
   half-updated PWA cache — the loud state TRIGGERS the SW update check alongside the reload
   prompt (wire through the registerSW update mechanism, NOT a bare location.reload, so the
   stale half gets replaced, not re-served; this also resolves the 1.1 "update UX"
   carry-forward). **Pins:** mismatch surfaces pre-job (no work accepted on a mismatched
   pair); match costs one message each way.
3. **FileFormat/FileSource: excise BOTH.** Format-level recall (`_applyTransformationRules`,
   `_saveFileFormatAndSource`) is superseded by the 2.3 columnName pool (FEAT-005 «no format
   entity», FEAT-011 revised; parity proven by the map-once-reimport E2E). FileSource goes
   too — format-coupled, zero-UI consumer; S3a (2.7) defines a LEAN source notion from the
   design bundle's actual needs. Kills the `lastUsed: Date.now()` residue — the determinism
   grep drops to 2 classified sites. **Two excision disciplines:** (1) COMPLETE + DECLARED —
   every ported-suite assertion that exercised format-recall is adapted with an explicit
   callout (hunk-audited, no silent deletions; QA will diff the suites); (2) grep-proven —
   zero FileFormat/FileSource references survive outside docs/changelog mentions.
4. **NFR-007: always worker (zero-threshold).** Production EngineClient is ALWAYS
   worker-backed; decode/import/parse never touch the UI thread; progress events flow for
   every job; the direct client remains for vitest (the harness rides the real worker-backed
   client post-sunset). **Pins:** (1) worker spawned ONCE at app boot (~50–150ms amortized
   into startup), persistent, respawn-on-death — the drain semantics make death loud, not
   hung; (2) "UI diff ≈ zero": switching Direct→Worker transport in production changes only
   the wiring point (the 1.1 seam pays off — assert it). **KB note:** NFR-007 gets the
   clarification — <1s-on-laptop is a perceptual BUDGET; implementation = always-worker (the
   main-thread-parsing mandate is removed).

## 1. The serializable session protocol (convention-derived — stated for veto)

The stage objects (RxJS BehaviorSubjects, methods, Message instances) CANNOT hop threads.
The client surface is therefore a session protocol over plain data:

- **The worker owns the object graph.** `ImportStatementServiceImpl` + stages live worker-side
  in a session registry keyed by `sessionId` (one active import session per client is the
  product reality; the registry supports N for tests).
- **The client speaks DTOs.** New `EngineClient` methods (each bumps nothing yet — they ARE
  contract v2, the first real bump):
  - `decode(input) → DecodeResultDTO` (bytes in via transferable ArrayBuffer; the 2.1 result
    is already plain data).
  - `importStart(rows) → { sessionId, stage2: Stage2SnapshotDTO }` (decode → stage1 → stage2
    happen worker-side in one hop; stage1 is not separately addressable — YAGNI, no UI needs
    it).
  - `importApplyColumn(sessionId, columnId, definition, params) → Stage2SnapshotDTO | ColumnRejectionDTO`
    (the 2.4 `ColumnTransformRejection` serializes: counts, threshold, cellErrors with
    rowIndex+columnId+message-key).
  - `importResetColumn(sessionId, columnId) → Stage2SnapshotDTO` ·
    `importConfirmRecall(sessionId, columnId)` (GUESSED→confirmed) ·
    `importResolveCollision(sessionId, confirm)` (LWW confirmSave path).
  - `importNext(sessionId) → { rows, rowErrors, skipped } as DTOs | UnmappedColumnsDTO`.
  - `importAbort(sessionId)` — frees the worker-side session.
  - `importGetRows(sessionId, offset, count) → RowWindowDTO` — windowed row access (S3b
    renders a scrolling table anyway, NFR-008).
- **Session lifecycle (founder refinement 1, explicit in contract v2):** aborted/completed
  sessions free the worker-side stage graph (no leaks). ONE active session at a time (the
  wizard is a single flow): `importStart` while another session is active → loud structured
  `SessionAlreadyActiveError` — the caller aborts first, explicitly (PINNED). Sessions die
  with the worker (decision-1 semantics); after respawn, `importStart` is the only way
  forward — no session resurrection. (2.7's useBlocker exit-protection and the wizard reset
  path call `importAbort`.)
- **Snapshot row economy (founder refinement 2):** snapshots do NOT re-ship the row set.
  Split: the column-state snapshot (cheap — column defs/states, N-of-M, collision, unmapped
  list, per-column SAMPLE cells capped at the 2.2 sample size) returns from every apply;
  full row data flows ONLY through windowed `importGetRows`. **PIN: applyColumn cost does
  not scale with total row count on the wire** (asserted: snapshot byte-size for a 10k-row
  session ≈ that of a 12-row session, modulo column count).
- Snapshots carry: columns (id, originalName text, definition, params, recallState,
    sample cells with error/ignore message KEYS), N-of-M, `lastSaveCollision`,
    unmapped list. Message objects serialize as their catalog keys + params (the established
    $t convention — the UI catalog renders at 2.7/2.8).
- **Out-of-band events** (worker→client, no request id): `progress {jobId, phase, done, total}`
  (HC-10 honest counts), `blocked` (IDB onblocked → the loud multi-tab state), `dead`
  (respawn notification). Client exposes them as `EngineClient.onEvent(cb)` — a plain
  callback, NO RxJS on the public surface (NFR-003 boundary holds).

## 2. Transport hardening (carry-in #1, production WorkerTransport)

- Pending-map drain: `worker.onerror` / explicit `terminate()` / respawn → EVERY pending
  request rejects with a structured `EngineWorkerDiedError {phase, jobsLost}` — no hung
  promises (acceptance: kill mid-job → structured failure).
- Per-request timeouts: default per method class (handshake 5s; decode/import-RPC 30s);
  long jobs stay live via progress events resetting the liveness window (a job that emits
  progress is never timed out; one that goes silent past its window rejects loudly).
- Structured error serialization: worker-side errors cross as
  `{name, messageKey, params, payload}` — typed rehydration client-side
  (`ColumnTransformRejection`, `UnmappedColumnsError`, `BaseCurrencyNotSetError`,
  `ContractMismatchError`, `EngineWorkerDiedError`); unknown errors rehydrate as
  `EngineError` with the original name preserved (HC-7: never silently mangled).

## 5. apps/web changes (checkpoint (c): UI diff ≈ zero)

`engine.ts` wiring point: `createDirectEngineClient(...)` → worker spawn + handshake +
`createWorkerEngineClient(...)`. The ONLY other UI change: a minimal chrome-level status
surface for the three loud states (blocked / contract-mismatch+update / worker-died) — small,
presentational, altus-styled; full states matrix is 2.7/2.8's. Everything else byte-unchanged
(assert: web diff ≈ wiring + the status surface + harness switch).

## 6. Tests

Transport: drain-on-error/terminate (no hung promises) · per-request timeout fires ·
progress resets liveness · handshake match/mismatch (mismatch pre-job, pinned) · structured
rehydration per error type · respawn-on-death loud. Worker host (via @vitest/web-worker, the
1.1 pattern): full session protocol over a REAL worker hop — decode→importStart→applyColumn
(incl. rejection DTO)→next→typed-row DTOs; recall prefill + N-of-M over the hop; engine-config
hydration live (store override bites over the hop). Mid-job kill atomicity pin (decision 1).
Excision: suites adapted with declared callouts; grep-proof test (no FileFormat/FileSource
identifiers). Boundary spec: updated runtime keys + `./qa` import now FAILS (the sunset
asserted); fence teeth-check. Build: chunk-split assertions on the manifest + precache list;
offline parse (preview + SW, scripted like the 1.1 offline proof). Real statements local spec:
unchanged counts through the WORKER-backed path (one E2E variant over the hop).
