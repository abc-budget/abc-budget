## Goal

Bring `abc-altus.css` into the app as the **single source of design tokens**, vendor the fonts and
brand assets, and build the **shared ALTUS component library** (keys, lamps, badges, panels/logchip,
cream/teal/CRT/paper surfaces, dot-matrix gauge, brand mark, icon set) plus a **QA gallery page**
that renders everything at real sizes for pixel-wise comparison against the bundle.

## Bundle facts (verified by reading the source)

- `abc-altus.css` (152 lines): tokens (`--teal/--beige/--gold/...`, `--f-*`, `--r-*`, `--sp-*`,
  shadows) + atom classes (`.panel/.panel-h/.panel-b/.logchip`, `.cream`, `.key` gold/orange/green/
  beige/ebony + `.sm`/`.pressed`, `.lamp` green/gold/orange/gray/off, `.badge` within/almost/over/
  muted, `.chip/.codechip`, `.crt`, `.paper`, `.screw`, type helpers, `hr.dash`).
- **Brand mark** = static `assets/abc-flap-mark.svg` + Jost wordmark («ABC Budget») — not an
  animated split-flap component.
- **Dot-matrix readout** = the prototype `Gauge` (28 lit cells + % footer + over-limit row +
  archived state); its CSS lives in per-prototype `<style>` blocks, NOT in `abc-altus.css` — the
  library extracts those rules into our own token-based component CSS.
- **State badge** = `.badge` + inline SVG + label — the §4 pattern, exemplified in prototype JSX
  (`BADGE_ICON` within/almost/over/muted/history).
- `abc-iconset.jsx` = ~52 category glyphs in 8 groups (1.6px single-weight stroke, 24×24) **plus** a
  `CURRENCIES` array + helpers — the latter collides with 1.6's engine-owned currency reference.
- Fonts: 11 TTFs (Jost ×4 weights, IBM Plex Sans ×3, IBM Plex Mono ×4), self-hosted via
  `@font-face` in `abc-altus.css`.

## Decisions (resolved with the founder)

1. **Placement:** `apps/web/src/ui/altus/` — in-app module, no workspace package (single consumer;
   NFR-003 is engine⇄ui, not ui⇄ui). Two disciplines keep a later `packages/ui` extraction
   mechanical: (a) the token layer stays **verbatim-traceable** to the bundle; (b) `altus/` stays
   **app-agnostic** — props only, no engine/app imports (ESLint-enforced).
2. **Gallery:** second Vite entry **`/gallery.html`** (multi-page build). QA chrome reachable by
   URL only, never linked from the app (not product IA per FEAT-030). Ships in dist — acceptance is
   verified on the BUILT artifact. Firebase SPA rewrite is a non-issue (existing files are served
   before rewrites). Workbox precaching it is harmless.
3. **Iconset:** **icons only.** The ~52 glyphs + 8 groups become a typed ES module preserving the
   1.6px-stroke 24×24 geometry verbatim, **including per-glyph uk/en names + group ids** (icon-
   collection metadata, not currency data). `CURRENCIES`/`curOf`/`curName`/`BASE_ALIAS` do **not**
   enter the codebase — currency display wires through the engine at 1.6 (ENT-019,
   `@abc-budget/currencies`); the «base» alias UI arrives with its consumer (EP-7). Currency-looking
   gallery samples are inline fixtures, labeled as fixtures.

## 2. CSS strategy — two layers, one diffable

- **`altus.css`** is a clean derivative of the bundle: byte-equal **except** the `@font-face`
  `url()` paths (→ `/fonts/...`). A 2-line header comment documents the exact delta
  («derived from abc-altus.css 2026-06-10; only @font-face url() paths changed») so the
  diffability claim is self-documenting for future designer re-exports *(founder note #1)*.
- **`altus-components.css`** is ours: every rule the prototypes kept in per-page `<style>` blocks
  that the library needs (gauge `.cell` grid + foot, `.brand-name`, …), rewritten to read **only
  `--*` tokens** — no hex/px literal where a token exists.
- Both are imported once per entry (app + gallery). Token acceptance ("not hardcoded
  per-component") holds by construction: components carry classes, not inline styles
  (the one exception: `Gauge`'s lit-cell colour, which the prototype itself sets inline from state
  meta — it reads the state's token-derived colour).

## 3. Components — thin, app-agnostic wrappers

- Props only; no engine imports; no app state; no screen logic. **ESLint:** a
  `no-restricted-imports` block for `apps/web/src/ui/altus/**` bans `@abc-budget/engine` and
  relative escapes into app modules.
- **`StateBadge` takes its label as a prop** — the library is translation-agnostic (i18n lands in
  1.4); it has **no colour-only mode**: the icon is part of the state definition (§4 baked into the
  API). `Lamp` is exported standalone but the gallery demonstrates the `lamp + word` pairing per
  the report.
- IBM Plex Mono for amounts/ISO codes/chrome/readouts comes from the bundle's own classes
  (`.amount`, `.mono-s`, `.eyebrow`, `.crt`, `.logchip`, `.codechip`) — the library exposes them
  as-is; no re-invention.
- Component internals are **ours** (PROVENANCE: match visual output, not prototype structure);
  prototype JSX is consulted for markup shape only where fidelity demands it (Gauge cells, badge
  icon paths).

## 4. Gallery (the acceptance artifact)

`gallery.html` → own React root. Sections:
type specimens (eyebrow / h-display / h-sec / body-p / mono-s / amount) · keys (all variants ×
default/pressed/sm × with/without icon) · lamps (+word pairings) · badges (all 5 states) ·
panels + logchip + screws · cream/CRT/paper surfaces · Gauge (within/almost/over/archived/empty) ·
BrandMark · all ~52 icons at **16px and 24px** in their 8 groups with uk/en names.
Currency-looking strings = inline fixtures, labeled.

## 5. Verification

- **Dev pre-verification:** Playwright opens bundle prototypes and the gallery side-by-side at
  identical viewport, screenshots both; key atoms (key gold, badge over, panel header, gauge,
  brand) compared visually before handoff.
- **PM does the formal pixel eyeball at the final DoD** (side-by-side vs the bundle prototypes).
- **Automated pixel-diff CI is NOT built** (YAGNI — the design is final).
- **Tests (vitest, jsdom):** render-tests per component — class/DOM assertions; the §4 invariant
  (every `StateBadge` renders an `svg` + non-empty label); Gauge cell math (lit count per ratio,
  over-limit row, archived); icons module integrity (52 glyphs, 8 groups, every glyph has uk+en
  names). No screenshot tests.

## 6. Out of scope / guardrails

- No screens (S1–S4 layouts come with their epics) · **no review state-switcher** (PROVENANCE /
  FEAT-030) · no `CURRENCIES`/`curOf` (→ 1.6) · no i18n wiring (→ 1.4; gallery hardcodes uk/en
  specimen strings) · no theme switching (single palette by design) · no Storybook (the gallery IS
  the gallery).
- Engine, barrel, boundary tests: untouched.

## 7. Carry-forward

- **Fonts → EP-9.1 (perf budget):** 11 vendored TTFs ≈ ~1MB in the Workbox precache — acceptable
  now; tag a woff2-subset optimization pass for EP-9.1. Do **not** convert now: verbatim-from-bundle
  wins at this stage *(founder note #2)*.
- Designer re-export procedure: diff bundle `abc-altus.css` → repo `altus.css` (only the documented
  delta should appear), re-diff icon geometry if `abc-iconset.jsx` changed.
- `packages/ui` extraction: remains mechanical thanks to disciplines (a)+(b) — revisit only if a
  second consumer (Plus shell) materializes.

## Definition of Done (what the PM verifies — the ONE checkpoint)

1. `pnpm install && pnpm typecheck && pnpm lint && pnpm test && pnpm build` all clean.
2. `/gallery.html` in the **built** artifact renders the full library: keys/lamps/badges/panels/
   surfaces/gauge/brand/icons (16px + 24px rows), matching the bundle **pixel-wise**
   (PM eyeball, side-by-side vs prototypes; dev's Playwright pre-verification done before handoff).
3. Tokens not hardcoded per-component: `altus.css` is verbatim-traceable (documented 2-line delta);
   `altus-components.css` reads tokens only.
4. IBM Plex Mono used for amounts/ISO codes/chrome/readouts (per the design classes).
5. State-not-colour-alone: `StateBadge` API has no colour-only mode (asserted by test).
6. No review state-switcher anywhere; no CURRENCIES data in the codebase; engine/barrel untouched.
