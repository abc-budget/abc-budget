  EngineClient wire methods.** The graph is ALREADY composed (`composition-root.ts`) — only the wire surface

### The 4.9a wire surface (the v4 additions) — function signatures
All session-scoped methods take the live `sessionId` (S3c reuses S3b's session — `importNext` no longer
frees it). New DTOs are plain serializable shapes (no engine internals cross the wire).
```ts
// OPS/ — windowed rows + their live category + the draft-filter preview + the live-match count.
importCategorizedRows(sessionId, { offset, count, segment: 'all'|'uncat', draft?: ConditionDTO[] /*4.9c: dumpCategoryId?*/ })
  : Promise<{ rows: CategorizedRowDTO[]; total: number; matchCount: number }>;
//   CategorizedRowDTO = { rowIndex; date; amount; currency; description; counterparty; account;
//                         bankCategory; mcc: number|null; categoryId: string|null; isManual: 0|1;
//                         ruleId: number|null /*winning rule, for LOG routing*/ }
//   (4.9b will add optional `previousCategoryId`; 4.9c optional `atypical` — the DTO + cell are shaped for it.)
importConditionFields(sessionId): Promise<ConditionFieldDTO[]>;   // the import's MAPPED columns (ENT-009/010)
//   ConditionFieldDTO = { field: TypicalityField|'date'|'account'; valueKind: 'text'|'num'|'num2'|'day'|'day2'|'code'|'optone'|'optset'|'regex'|'bool'; operators: string[]; options?: {value,label}[] }
importWhy(sessionId, rowIndex): Promise<WhyTreeDTO>;              // LOG/ — the 4.2 debugger path
//   WhyTreeDTO = { manual: { categoryId } | null; rules: { ruleId; status:'win'|'miss'|'neutral';
//                  conditions: { field; operator; value; met: boolean|null }[]; categoryId }[]; winnerRuleId: number|null }
importRulesList(sessionId): Promise<RuleSummaryDTO[]>;            // RUL/ list (first-match order + applied count)
//   RuleSummaryDTO = { ruleId; conditions: ConditionDTO[]; categoryId; appliedCount /*rows THIS import won*/ }
rulesCreate(conditions: ConditionDTO[], categoryId): Promise<{ ruleId: number }>;  // live append (4.3b create+persist)
categoriesList(): Promise<CategoryDTO[]>;                          // CategoryDTO = { id; name; icon; currency /*«base» resolved*/ }
categoriesCreate({ name, icon, currency }): Promise<CategoryDTO>; // 4.3a createFromSearch / create
```
**`ConditionDTO = { field: string; operator: string; value: unknown; currency?: string }`** — the
serializable condition. **Worker-side `conditionsToRules(ConditionDTO[]): Rule[]`** maps it to the 4.1
factories; **amount goes through `createAmountCondition(op, currency)`** (the bare factory is private; the
DTO carries the paired `currency` → the UI cannot save an amount condition without one). This mapping +
`saveDecisionTree`/`create` live in a new worker-side **categorization client method group** on the engine
service (composed from the already-built EP-4 pieces).
