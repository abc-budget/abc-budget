## The principle (what the algorithm must NOT do)
NEVER judge a rule "bad" or a set "mixed." Similarity is the USER'S FILTER. The tool ranks structural
atypicality **WITHIN a bucket** (baseline = the bucket itself), surfacing the least-typical for the user to
self-validate their own false positives (a `description contains "market"` grocery filter silently catching
"STOCK MARKET FEE"). **Hint, not judge** — never blocks, dismissable. The hard line: a **homogeneous bucket
→ EMPTY tail** (zero false alarms). That is the headline acceptance test and it drives every math choice below.

## The math (per bucket, after the N-gate)

### Guard 1 — N-gate
`bucketSize < N_min` → `{ skipped: true, flagged: [] }` (show the raw set; never manufacture an outlier from
a tiny sample). **N_min = 8.** Justification: the median + **MAD** stabilize around n ≈ 7–8; a 1-of-8
minority (12.5%) is distinguishable from a 1-of-3 coincidence (33%). Tunable const. (Amount has an extra
per-currency floor — below.)

### Guard 2 — field informativeness (which fields may contribute)
A field contributes ONLY if it has a **typical pattern most of the bucket shares** — else it can't define
"atypical" and is dropped (so a uniform-diverse field never makes "everyone an outlier", and a constant
field is a no-op). Per field, `informative_f` (boolean) AND it is excluded if it is a filtered categorical
field:
- **categorical** (mcc / counterparty / currency / bankCategory): `coverage_f ≥ 0.5` AND a **dominant mode**
  exists `p_mode ≥ 0.5` (over PRESENT values). No dominant mode (uniform-diverse) → not informative → dropped.
- **amount**: per **currency** sub-bucket with `≥ 5` present values (robust stats need a floor). A currency
  with `< 5` → amount not informative for ITS ops (never cross-currency).
- **text** (description): informative if a **core vocabulary** exists — at least one non-filter token with
  document-frequency `df ≥ 0.5`. All-distinct text (no shared core) → not informative → dropped.
`coverage_f` = fraction of the bucket where the field is present (non-null / non-empty). Absent in ALL →
coverage 0 → dropped. (Absent in ONE = a minority = a signal, preserved.)

### Step 4/5 — per-field atypicality `a_f(op) ∈ [0,1]` (0 = typical, 1 = maximally atypical)
- **Categorical:** `a_cat = clamp(1 − p(op.value) / p_mode, 0, 1)`, where `p(v)` = bucket frequency of `v`,
  `p_mode` = the mode's frequency. The **mode → 0**; a value at half the mode's frequency → 0.5; a singleton
  → ≈1. Homogeneous (all = mode, `p_mode = 1`) → **0 for all**.
- **Amount (per currency):** robust z `= |amt − median_c| / (1.4826 · MAD_c)` (1.4826 makes MAD a σ-estimate;
  median+MAD are outlier-robust — NOT mean/stdev). `a_amt = clamp((z − Z0) / (Zmax − Z0), 0, 1)`, **Z0 = 3**
  (below 3 → 0), **Zmax = 6** (≥6 → 1). `MAD_c = 0` (constant amounts): `amt = median → 0`; `amt ≠ median →
  1` (a lone different amount in an otherwise-constant bucket IS the outlier). Homogeneous amounts → 0 for all.
- **Text:** tokenize `description` (lowercase, split on non-alphanumeric, drop the rule's filter tokens).
  `df(t)` = fraction of bucket ops containing token `t`. `a_text = max over op's non-filter tokens t of
  (1 − df(t))` — the op's **rarest** token drives it (a token nobody else has → ≈1). All tokens common
  (`df = 1`) → 0. Homogeneous text → 0 for all. (The rare tokens are reported in the attribution.)

### Step 5 — cross-signal combination (the crux) — **NOISY-OR, not a weighted average**
`atypicality(op) = 1 − Π_{f informative & present for op} (1 − a_f(op))`.
- **Why NOT the literal "weighted sum / average":** a weighted AVERAGE dilutes a single strong signal — the
  STOCK-MARKET-FEE op (MCC outlier `a_mcc ≈ 1`, but amount/text typical `≈ 0`) averages to ≈ 0.3 and would
  **fail the headline test**. ENT-021 ranks MCC the STRONGEST signal precisely because it can flag alone.
  Noisy-OR lets ONE strong field flag (1 − (1−1) = 1) **and** still accumulates several weak signals, while
  preserving the homogeneity guarantee. **This is the one place I depart from the literal "weighted sum" in
  the prompt — flagging it; the dilution flaw is real.** (Alt considered: an un-normalized `Σ w_f a_f` with
  an absolute cutoff — also undilutive; I prefer noisy-OR for the clean [0,1] range + the zero-false-alarm proof.)
- Field weight in the product = the informative gate (binary) × coverage; no per-field "signal-strength
  prior" in the SCORE (kept lean, fewer magic constants) — the ENT-021 signal ranking (MCC > amount > text >
  categorical) orders the **attribution reasons**, not the number.

### Step 6 — tail = ABSOLUTE threshold (NOT top-K)
`flagged = ops with atypicality ≥ T_abs`, sorted DESC. **T_abs = 0.6** (tunable).
**Zero-false-alarm guarantee (the proof):** in a homogeneous bucket every `a_f = 0` (every value is the mode,
every amount = the median with z = 0, every token has df = 1), so every product term is `(1 − 0) = 1`,
`atypicality = 1 − 1 = 0 < T_abs` for **every** op → **empty tail**. This holds for ANY `T_abs > 0` — the
guarantee is structural (homogeneity ⇒ all a_f = 0), independent of the threshold value. Top-K would always
surface K rows → it is explicitly rejected.

### Step 7 — attribution
Per flagged op, for each field with `a_f ≥ 0.5` emit a `TypicalityReason`, ORDERED by the ENT-021 signal
strength (mcc → amount → text → counterparty/currency/bankCategory), then by `a_f` desc:
- categorical → `{ field, kind:'categorical-minority', value: op.value }` (mcc reports the code).
- amount → `{ field:'amount', kind:'amount-outlier', magnitude: round(z) }` (UI shows "≈ ×N the typical").
- text → `{ field:'description', kind:'rare-tokens', tokens: [the rarest non-filter tokens] }`.

## Determinism (HC-9)
Pure functions of the bucket; **no `Date`, no `Math.random`**. Median/MAD via a deterministic sort; histogram
+ BoW iteration in a fixed order; token order canonical (sorted). Final sort: by `atypicality` DESC, **tie-break
by `row.rowIndex` ASC** (stable, total order). Same bucket → byte-identical ranking + tail.

## Complexity
Each op is in ≤ 1 bucket (first-match). Per bucket: **profile once** (histograms O(n); BoW O(tokens);
median/MAD via sort O(n log n) per currency sub-bucket), then **score each op O(#fields)** against the
PRECOMPUTED profile — **no per-op re-scan of the bucket**. Overall ≈ O(N) (the only super-linear term is the
median sort, O(N log N) worst case; acceptable, deterministic).

## Acceptance / DoD (tests-with-teeth)
- atypical op → tail WITH structured reason (STOCK MARKET FEE in a grocery bucket → flagged, `mcc-minority` + the code).
- **HOMOGENEOUS bucket → EMPTY tail** (the headline; the a_f = 0 ⇒ atypicality 0 proof).
- N-gate: `bucket < 8` → skipped (raw set, no manufactured outlier).
- field-availability: a null/constant field contributes ZERO; the FILTERED field excluded; a uniform-diverse
  field does NOT make everyone an outlier.
- per-currency amount: a 10× amount in one currency flagged; a second currency NOT cross-contaminated.
- deterministic: same bucket → same ranking + tail, byte-stable.
- NOT on the Auto-Other / uncategorized set (only rule buckets).
- O(N): per-op scoring hits the precomputed profile, no bucket re-scan.
