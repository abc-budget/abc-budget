## The engine model (what we wire — already merged)
**4.6 (`internal/rules/auto-other.ts` + `categorize-with-overrides.ts`):** the L1→L4 ladder —
`effectiveCategory(row, ctx, dumpCategoryId)` = L1 manual → L2 override → L3 rule tree → **L4 dump** → null.
`remainderRows(rows, ctx, dumpCategoryId)` = rows whose effective category is null. `isComplete(...)` =
remainder empty. `autoCategorize` accepts `deps.dumpCategoryId?` and emits `isManual=0` for L4 rows (the dump
is TRANSIENT — never a rule, never a footprint-`isManual=1`, GONE on re-import). `lastRemainderCategoryId` =
`UserSettingsService.getSetting/setSetting(SettingKeys.LAST_REMAINDER_CATEGORY_ID)`. `computeRemainderMagnitude(
remainderRows, {ratesService, base})` → `{opCount, baseTotal, uncachedTail: Map<currency, amount>}` — cross-
currency converted at the OPERATION date; uncached rows fail-soft into the per-currency tail (the «≈» signal).
**4.8 (`internal/rules/typicality/`):** `rankBucket(rows, filteredFields): {skipped, bucketSize, flagged:
FlaggedOp[]}` — PURE/SYNC/moment-agnostic. `FlaggedOp = {row, atypicality, reasons: TypicalityReason[]}`.
`TypicalityReason = {field: TypicalityField, kind: 'categorical-minority'|'amount-outlier'|'rare-tokens',
value?, magnitude?, tokens?}`. Buckets via `bucketByWinningRule(rows, tree)` (first-match, keyed by ruleId,
**excludes the uncategorized remainder**) or `draftBucket(rows, complexRule)`. MCC→title is **web-side**
(`mccTitle(mcc, lang)`); the wire carries the raw mcc code in `value`. **None of 4.6/4.8 is exposed on the
worker `CategorizationService` yet** — all new v6 wire.

### Completeness-gate source [FORK — recommend window.remainderCount]
Add **`remainderCount` to `CategorizedWindowDTO`** — computed once per window over ALL rows (dump-aware:
`remainderRows(rows, ctx, dumpCategoryId).length`). The «Далі» gate = `remainderCount === 0`; «Решту → Інше»

separate `importCompleteness` call duplicates the categorization.*

### The class-killer continuity (the recurring id-drift lesson)
Export `TypicalityField` and `TypicalityReasonKind` (= `TypicalityReason['kind']`) **type-only** from the barrel
(exactly as `RuleOperatorId` in the id-drift fix). The UI's reason-chip renderer is keyed off the union (a
`switch` with an exhaustive `never` check, or a `Record<TypicalityReasonKind, …>`) → adding/renaming a 4.8 kind
fails `tsc`. **Kills the typicality-reason drift class proactively** — the bundle only illustrated `mcc`; the
real 4.8 `categorical-minority` spans **mcc + counterparty + currency + bankCategory**, so the chip mapping
MUST branch on `field`, not hardcode the prototype's 3 (the recurring teeth).
