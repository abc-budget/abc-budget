## ¬ßA ‚ÄĒ CATEGORY MODEL (4.3a)

**Module:** `internal/categories/` (the `Category` type already exists there from 4.2).
```
internal/categories/types.ts        # EXISTS (4.2). Adapt: id: string; icon (glyph-id); currency may be ¬ębase¬Ľ.
internal/categories/categories-dao.ts   # NEW: IDBDao<string, Category>, keyPath 'id' (NO autoIncrement)
internal/categories/categories-service.ts # NEW: create / list / get / pick + inline create-from-search
internal/persistence/engine-db.ts    # MIGRATION v6: createStore('categories', { keyPath:'id' }) + indexes
```

### Adapts (ENT-018)
- **id ‚Üí STRING.** keyPath `'id'`, **no autoIncrement**. **FORK 2 ‚ÄĒ id generation [recommend service-gen
  UUID]:** the service generates `crypto.randomUUID()` at create (it's a fresh PK, not a derived/hashed
  value ‚ÄĒ no HC-9 determinism concern). *Alt:* caller-supplied id. Aligns `footprint.categoryId: string`.
- **icon (FEAT-026, RISK-008):** drop `image`/`imageMetadata`; add `icon: string` (an ALTUS glyph-id). No
  image URL anywhere.
- **¬ębase¬Ľ living alias [FORK 3 ‚ÄĒ recommend store-literal-resolve-at-read]:** store the **literal sentinel
  `'base'`** in `currency`; the **service** resolves `'base'` ‚Üí the current base currency at READ
  (`get`/`getAll`/`pick`) via an injected base-currency resolver (engine settings). A category set to a
  concrete ISO keeps it. The DAO stays dumb (stores/returns the raw row). Sentinel const
  `BASE_CURRENCY_ALIAS = 'base'`. *(Prior-art baked a concrete ISO at save ‚ÄĒ we do NOT; the alias stays
  living so a later base-currency change follows.)*
- **Minimal surface (EP-4 needs only):** entity + `create` (incl. **inline create-from-search**, ENT-018:
  given a typed name with no match ‚Üí create + return it, auto-selected) + `list` + `get` + `pick`. The model
  **carries `isArchived`** but archive/unarchive **management UI + guarded effective-dated currency-change
  (ENT-020) = EP-7/7.1 ‚ÄĒ NOT built here.** (I'll port `archive`/`unarchive` DAO-level only if trivially
  needed by list filtering; otherwise leave for EP-7 ‚ÄĒ see DoD.)
- **EP-6 note (flag, don't build):** the TOTAL sentinel `categoryId = -1` becomes a **reserved STRING**
  (e.g. `'-1'`). Not built in 4.3; noted so the string-id space reserves it.

### Validation (ported, adapted)
`name` required (non-blank) ‚Üí else `InvalidCategoryError`; `currency` either a valid ISO (validated against
the currency reference) **or** the literal `'base'`; `isArchived` defaults false. No IoC ‚ÄĒ inject the
currency reference + base-currency resolver directly.

## ¬ßB ‚ÄĒ RULE PERSISTENCE (4.3b)

**Module:** `internal/rules/` (alongside the 4.2 evaluator).
```
internal/rules/complex-rule-dao.ts   # NEW: IDBBatchDao on 'complexRules'; ComplexRuleDTO + (de)serialize
internal/rules/rule-persistence.ts   # NEW: create / update / reorder + RxJS-FREE reload + saveDecisionTree
internal/persistence/engine-db.ts    # MIGRATION v7: createStore('complexRules', { keyPath:'id', autoIncrement }) + indexes order, categoryId
```
- **`complexRules` store:** keyPath `'id'`, **autoIncrement TRUE** ‚ÄĒ the **rule id stays a number** (only
  the *category* id became a string). Indexes: `order`, `categoryId`. **`ComplexRuleDTO = { id?: number,
  rules: RuleDTO[], categoryId: string, order: number }`** (categoryId STRING ‚ÄĒ the A‚ÜĒB seam). `RuleDTO =
  { field, operation }`; (de)serialize rebuilds `Rule`s via the 4.1 factories (drop the dropped `source`
  factory).
- **Persistence service (RxJS-FREE):**
  - `create(rules, category)` ‚Üí next order = max+1; DAO create.
  - `update(rule)` ‚Üí preserve existing order; DAO update.
  - `reorder(ruleIds)` ‚Üí uses **4.2's `reorderRuleOrders`** (identifyNewOrder + ID validation) to compute
    new orders, then DAO batchUpdate. **IDs preserved.**
  - **`reload(): Promise<DecisionTree>`** ‚ÄĒ re-read `getAllOrdered()` + resolve categories ‚Üí build a fresh
    `DecisionTreeImpl` **synchronously returned** (NO `BehaviorSubject`, no Observable). Callers call
    `reload()` after a write and use the returned tree. This is the RxJS-free replacement for the prior-art
    reactive `_decisionTreeSubject.next`.
  - **`saveDecisionTree(tree)` ‚ÄĒ the delta-ID-preservation apply path (4.5 reuses it):** diff the incoming
    tree against persisted rules; create new, update changed (PRESERVING ids), delete removed; preserve
    order. Ported RxJS-free.
- **NO override/sandbox/UI** (4.4/4.5).
