## Locked decisions (founder, 2026-06-12)

1. **Contract v3 — dedicated methods + pure export.** `EngineClient` += `getBaseCurrency():
   Promise<string|null>` (null = unset — the gate's probe; no exception-driven control flow)
   and `setBaseCurrency(iso)` (validates via the 1.6 reference, loud on invalid).
   `localeToCurrency` exports from the barrel as a PURE runtime function — declared boundary
   change: runtime keys = `[createDirectEngineClient, createWorkerEngineClient,
   localeToCurrency]`. `CONTRACT_VERSION → 3` (the bump rule's first real exercise; decision
   2's DTO change rides the same bump). NFR-009 holds: raw settings stay internal.
   **Pin — no stale-snapshot trap:** `setBaseCurrency()` → IMMEDIATELY `importStart` →
   `use_base` resolves to the just-set currency. (Base currency is read live from the DAO at
   resolution time, not from a pre-set snapshot — assert explicitly; the gate's whole flow is
   set-then-import in one breath.)
2. **Structural DATE errors — engine structural channel.** `GenerateRowsResult` (and DTO) +=
   `structuralErrors: Message[]`, detected BEFORE the row loop (no-DATE / multiple-DATE),
   ONE message per condition; individually-failing rows keep `rowErrors`. **Pins:** (a)
   TAXONOMY BOUNDARY — no-DATE and multiple-DATE are DISTINCT structural messages with
   different ДІЯ hints (map one vs unmap one), both pinned; (b) DECLARED MIGRATION — the
   2.4-era assertion («ignore-DATE → 12 per-row errors») becomes ONE structural message +
   zero row-error echoes; that E2E assertion updates as a NAMED change (the opposite of
   silent weakening — call it out for QA's diff).
3. **Base-currency dialog list — curated 8 + full reference.** Top optgroup = the prototype's
   8 (UAH USD EUR GBP PLN CHF CZK GEL, prototype order); below a divider the rest of the 1.6
   reference; localized names via the reference for BOTH groups (never hardcoded);
   auto-detect (`localeToCurrency`) preselects; native select with two optgroups (keyboard-
   typeable; search stays S4's picker pattern). **Pin — the lockout test:** a locale mapping
   OUTSIDE the curated 8 still completes first-run end-to-end (auto-detect preselects in the
   lower group).

  bundle's informational copy — no fake numbers).
- **Multi-sheet:** decode's `meta.otherSheets` surfaces as a neutral note if non-empty; any
  real multi-sheet need → Designer residual, flagged not invented (2.1 decision).

## 1. Architecture

```
apps/web/src/app/screens/import/
├─ ImportFlow.tsx            # MODIFY (1.5 shell): per-step body components; real canAdvance
│                            #   for step 1; useBlocker exit-protection → importAbort
├─ s3a/
│  ├─ S3aSource.tsx          # state machine: idle|decoding|recognized|unknown|error
│  ├─ DropZone.tsx           # pick/drop + sample entry (FEAT-001 two paths)
│  ├─ FileChip.tsx           # name/size/rows + replace/remove (→ importAbort)
│  ├─ RecognizedPanel.tsx    # N-of-M + savedmap rows + dedup copy + proceed note
│  ├─ UnknownPanel.tsx       # 0-of-M first-ever path (→ mapping, gold key)
│  ├─ ErrorPanel.tsx         # ЩО/ЧОМУ/ДІЯ from decode issues (HC-7 loud)
│  └─ BaseCurrencyDialog.tsx # cold-start gate (decision 3 list; modal per bundle)
└─ (s3b/s3c/s3d keep 1.5 placeholder bodies)
```

- **Gate #1 (`canAdvance`):** step 1 advances iff state ∈ {recognized, unknown} (both are
  legitimate proceed paths per FEAT-009 — review/proceed offered, never forced); idle,
  decoding, error → Next disabled. «Назад» → Dashboard (exists, FEAT-030).
- **Cold-start gate:** on entering /import, `getBaseCurrency()` === null → BaseCurrencyDialog
  BEFORE any file work; Confirm → `setBaseCurrency` → dialog closes, S3a proceeds; Cancel →
  navigate to Onboarding (the 1.5 route). Shown once (probe re-runs each entry; once set it
  never matches). Later changes = S4's guarded flow (out of scope).
- **Engine client:** the app's existing worker-backed client (2.6). Loud states (blocked/
  mismatch/died) already handled by EngineStatusBanner — S3a only adds the per-flow states.
- **i18n:** s3a-i18n.jsx keys → the app catalog (1.4 pattern), uk + en, incl. the typed-
  parameter messages (recogTitle(n,m) etc.).
