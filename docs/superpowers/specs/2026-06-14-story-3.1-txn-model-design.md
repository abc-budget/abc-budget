## 2. The one gap — display-label fallback (PM-ruled: add now)

ENT-006 specifies a display-label for list rendering (S3c/S3d): `description → counterparty → bankCategory → mcc → «—»` (first non-empty wins). The four source fields are all on the row, but **no helper computes the fallback** anywhere in the codebase (grep-verified). PM ruling (2026-06-14): add it now — fully ENT-006-specified, pure, returns a string; 3.1 (the txn-model story) is its natural home.

**Shape:**
- NEW `internal/importStatement/stage3/display-label.ts`: `displayLabel(row): string`.
- Input: a `Pick<TransactionRow, 'description' | 'counterparty' | 'bankCategory' | 'mcc'>` (only the fields the fallback reads).
- Logic: first **non-empty** of `description` → `counterparty` → `bankCategory` → `String(mcc)`, else `'—'` (em-dash, U+2014). "Non-empty" = a string that is non-empty after `.trim()`; `mcc` non-null → its string form.
- **No locale** (VIS-003: user content is not localized — the helper returns raw field text or the em-dash; it does not translate).

**Hard guards (PM):**
1. **DERIVED, NEVER a stored field.** `displayLabel` is compute-on-read; it must NEVER enter the footprint (HC-2/3, ENT-001 — the footprint keeps ONLY `{year, month, amountUSD, categoryId, hash}`). It is not added as a column on `ImportStatementStage3Row`.
2. **No persistence, no contract bump** in 3.1 (no DTO/`CONTRACT_VERSION`/wire change; pure internal helper).

## 3. Tests (TDD)

`display-label.spec.ts` covers the WHOLE fallback chain:
- description present → wins.
- description empty/whitespace → counterparty wins.
- description + counterparty empty → bankCategory wins.
- only mcc present → `String(mcc)` wins.
- all four empty/null → `'—'`.
- whitespace-only strings count as empty at each level.
- mcc `0` vs `null` (null → skip; a number → its string form).
