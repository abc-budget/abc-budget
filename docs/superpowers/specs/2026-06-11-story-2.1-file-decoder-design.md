## Goal

The decoder that does not exist in prior art: `decode(input) → { rows, issues, meta }` — any messy
real bank statement in, typed-ready raw rows out, **never silently**. Feeds the (to-be-ported)
stage pipeline, whose stage-1 contract is `Record<string, unknown>[]` (verified in prior art).

## Locked decisions

1. **Spreadsheet library: SheetJS CE via the official CDN tarball** (`https://cdn.sheetjs.com/...`,
   exact version pinned — same distribution the prior-art webapp used at 0.20.3). Reads BOTH legacy
   `.xls` (BIFF — real old bank exports) and `.xlsx`; built-in codepage/cp1251 support; Apache-2.0.
   **Three founder disciplines:** (a) PIN + integrity — exact tarball version in package.json,
   pnpm-lock records the hash (supply-chain pinned, never "latest"); (b) DOCUMENT the why next to
   the dep + in the decoder README (npm-registry `xlsx` is years-stale with known CVEs;
   cdn.sheetjs.com IS the official distribution; installs tokenless — no GitHub-Packages-style
   friction); (c) LAZY at the decode boundary — dynamic `import('xlsx')` only when a spreadsheet is
   picked; the chunk still lands in the Workbox precache (offline import is local-first-mandatory) —
   **DoD: airplane-mode decode of an .xls succeeds after first visit.**
2. **CSV: our OWN tolerant parser** (founder-locked). The wedge logic is the product — full control
   of issue collection, recovery behavior, and determinism. No PapaParse.
3. **No `EngineClient` growth in 2.1** — decoder is engine-internal; the client surface grows at
   2.6 with the contract-bump rules (epic plan).

## 1. Module layout (engine-internal)

```
packages/engine/src/internal/ingest/
├─ decode.ts            # decode(input: DecodeInput): Promise<DecodeResult> — the single entry
├─ types.ts             # DecodeInput { bytes: ArrayBuffer; fileName: string }
│                       # DecodeResult { rows: Record<string, unknown>[]; issues: DecodeIssue[]; meta: DecodeMeta }
│                       # DecodeIssue (ЩО/ЧОМУ/ДІЯ): { row: number; column?: string|number; what: string;
│                       #   why: string; raw?: string; action: DecodeAction } — coordinates always present
│                       #   (file-level issues use row: -1); DecodeAction = 'skipped-row' | 'kept-raw' |
│                       #   'padded-row' | 'truncated-row' | 'recovered-quote' | 'file-unreadable' | …
│                       # DecodeMeta { format: 'csv'|'xlsx'|'xls'; encoding?: 'utf-8'|'windows-1251';
│                       #   bom?: boolean; delimiter?: ','|';'|'\t'; headerRow: number; sheet?: string;
│                       #   otherSheets?: string[]; totalRows: number; decodedRows: number }
├─ encoding.ts          # bytes→string: BOM check → strict UTF-8 validation pass → cp1251 fallback
│                       #   (+ Cyrillic-frequency sanity score; ambiguous → UTF-8 + an issue)
├─ csv-parser.ts        # OUR tolerant CSV: delimiter sniffing (;/,/tab scored over first N non-empty
│                       #   lines), quote-state machine that RECOVERS from quote-in-field (re-sync at
│                       #   next delimiter/newline + issue), ragged rows padded/truncated + issue
├─ header-detect.ts     # preamble skip: window scan, header-row scoring (distinct non-empty strings,
│                       #   non-numeric majority, below-row width consistency); junk/summary/placeholder
│                       #   rows (incl. the real « - » row) → skipped + issue with coordinates
├─ sheet-decoder.ts     # XLS/XLSX via lazy import('xlsx') → matrix → the SAME header-detect path as
│                       #   CSV (one messiness pipeline, two front-ends); FIRST sheet decoded,
│                       #   meta.otherSheets lists the rest (see carry-forward)
├─ fixtures/            # synthetic set + generate.mjs + README.md (mapping fixture → messiness)
└─ *.spec.ts            # per-module + decode-level snapshot tests + the local real-statement spec
```

## 2. Contract semantics

- **Never an unstructured exception on content.** Every anomaly → `DecodeIssue` with coordinates,
  reason, raw value (the ЩО/ЧОМУ/ДІЯ shape that feeds S3b's UI). A partially-decodable file decodes
  partially and SAYS so (`meta.totalRows` vs `decodedRows` + issues). Corrupted bytes → `rows: []`
  + one file-level issue (`action: 'file-unreadable'`). Zero-byte input → same pattern, not a throw.
- **Raw fidelity (HC-2/3):** dual-currency cells, `—` placeholders, unparsed numbers pass through as
  raw values — typing them is stage-2/Story 2.2. Rows are transient; nothing persists in 2.1.
- **Determinism (HC-9):** no Date.now/random anywhere in the decode path; identical bytes → identical
  result, byte-for-byte (snapshot-tested).
- Row keys come from the detected header (deduplicated with a deterministic suffix on collision +
  issue); columns beyond the header width get positional keys + issue.

## 4. Tests & acceptance

- Per-fixture **snapshot tests**: full `{rows, issues, meta}` snapshots — correct rows AND complete
  issue lists.
- Unit suites: encoding (table-driven byte cases), delimiter scoring, quote-recovery state machine,
  header scoring/preamble skip, key dedup.
- **Fuzz case:** seeded corrupted bytes through `decode()` → structured issue, never an unstructured
  exception.
- **Local real-statement spec** (`real-statements.local.spec.ts`): auto-skips when
  `D:\abc-budget\mono_*.csv`/`ukrsib.xlsx` are absent (never CI, never committed). *(Founder note #2):*
  where stable, it asserts **hard local constants** — decoded row count per file, header names,
  spot-checked cell values — a count regression FAILS the local run; printed spot-checks are
  additional output for the QA protocol doc.
- **Airplane-mode .xls check** (DoD): after first visit, offline decode of a spreadsheet succeeds
  (lazy chunk precached) — dev Playwright pre-verification + QA manual pass.

## 5. Out of scope / carry-forward

- **Multi-sheet UX:** 2.1 decodes the first sheet + lists others in meta. The S3a design bundle has
  NO sheet-picker — if real bank .xlsx files turn out multi-sheet at 2.7, that's a **Designer
  residual** (flag it then; don't invent UI now) *(founder note #1)*.
- Typing/parsing of values (DATE/AMOUNT/…) → 2.2 (ported column.ts); recall → 2.3; thresholds → 2.4;
  pseudo-ops → 2.5; worker host + client surface growth → 2.6; S3a/S3b UI → 2.7/2.8.
- PapaParse fallback — only if our parser hits an unforeseen wall (decision reopens, founder).
- **From the final review (minor, deferred):** `bare-quote` issues fire per quote-character (noisy on
  preamble lines with quoted names — consider per-field aggregation when 2.8 renders issues) ·
  `totalRows` semantics differ subtly between paths (CSV drops all blank lines; sheets drop only
  trailing — document or unify when S3b surfaces the counts) · **⚠ 2.2 carry-forward:** a true
  date-serial xlsx cell (`t:'d'`/numeric+date-format) would arrive via SSF formatting, not source
  text — no current fixture/real file hits it, but 2.2's date typer must decide the policy ·
  `.xls`-named file with PK magic decodes silently without an extension-mismatch issue (edge).
