## Goal

S3b implemented as wizard **step 2** over the existing 2.6 session API: a raw mapping table (per-column sample cells) + per-column type mapping (instant dropdown / «More» 2-step with the scrollable markdown help panel), the loud UNKNOWN gate (`canAdvance()` #2, Option A), loud >30% rollback rendering, and the importNext worker-progress takeover on large files. **RECALL-ONLY:** type prefill comes exclusively from the recall pool — zero auto-detect/heuristics. Plus the one authorized engine-internal change: **defer recall-pool writes from apply to advance** (decision #4), so an abandoned session never pollutes the pool.

**Verified-finding rulings (PM, this brainstorming round):**
- **#4 — defer-commit to advance (engine change authorized).** VERIFIED: `recallPool.save()` fires fire-and-forget inside `stage2.applyColumn` (`implementation.ts:340`), at apply time — so an abandoned session pollutes the pool, violating "abandoned must NOT pollute". RULING: applyColumn STAGES the mapping in-memory on the session + still DETECTS collisions at apply-time (map-time UX unchanged); the actual pool WRITE happens on `importNext` (advance = the user's endorsement); `importAbort` DISCARDS staged writes; `importResetColumn` unstages. Staging lives in the shared stage2/session layer so Direct (tests) and Worker (prod) inherit identical behavior. **NOT a wire change — `importApplyColumn`'s signature and `CONTRACT_VERSION` are untouched.** The role fence is lifted for THIS scoped engine-internal change only.
- **#7 — sampleCells table (no engine change).** "windowed via importGetRows" was a spec slip: `importGetRows` returns stage3 typed rows (needs mapping complete, emits the no-DATE structuralError). The mapping table sources its per-column cells from `Stage2SnapshotDTO.sampleCells` (cap 10, evenly-spaced, value + error/ignore message keys) — already on every importStart/ApplyColumn/ResetColumn. `importGetRows` stays S3d/Review's typed-row reader. No raw-windowing method, no contract bump.
- **#8 — progress on importNext (no engine change).** Progress events fire only from decode + importNext. The full-screen WorkerPanel renders during the `importNext` ADVANCE on large files (its existing progress events: generate typed rows + pseudo-ops + SHA-256). Per-column applyColumn is sub-100ms, worker-hosted, progress-less → an inline-light pending affordance only. No progress from parseGeneric (HC-10: no theater).

## 1. The authorized engine-internal change (decision #4 — staged recall, flush-on-advance)

Shared stage2/session layer (`internal/importStatement/stage2/implementation.ts` + the recall pool):

- **Staging buffer** on `ImportStatementStage2Impl`: `_stagedRecallWrites: Map<normalizedName, { name, definition, params }>` (last apply per column wins).
- **`applyColumn`** (when a recall pool is mounted and `definition !== null`): replace the immediate `_recallPool.save(...)` write with (1) a **read-only collision DETECT** against the pool (`getEntry` + the existing type-change/params-change comparison) → set/clear `lastSaveCollision` exactly as today (map-time UX byte-identical); (2) **stage** the mapping in the buffer. NO IDB write. Stays non-blocking + loud-on-failure (HC-7).
- **`flushRecallWrites()`** (new stage2 method): commit each staged entry to the pool, preserving today's write semantics — new/identical → `save()` (writes); user-confirmed collision → `confirmSave()` (LWW); unresolved params-change → no write (keeps the stored entry, exactly as save() does today). Returns after all writes settle.
- **`unstageRecallWrite(columnId)`**: drop a column's staged entry (for `importResetColumn`).
- **Session wiring (direct-client; worker-host shims over it → parity by construction):** `importNext` calls `flushRecallWrites()` (alongside the heavy `generateForSession` that already emits progress — one advance, one commit, one honest takeover); `importAbort` frees the session WITHOUT flushing (buffer dies with the stage2 instance → discard); `importResetColumn` calls `unstageRecallWrite`.
- **Rejection ordering** (pin e): the >30% `ColumnTransformRejection` is thrown inside applyColumn's transform BEFORE staging — a rejected column never stages and stays UNKNOWN.

**Verification pins (a)–(g), all tested:** (a) apply→abort→next importStart ⇒ NO recall; (b) apply→advance→next importStart ⇒ recalled; (c) collision still fires at apply-time on a staged map; (d) importResetColumn discards a staged map cleanly; (e) >30% rejection still at apply, column→UNKNOWN, not staged; (f) HC-9 determinism preserved; (g) Direct ≡ Worker parity.

## 2. UI architecture (apps/web)

```
apps/web/src/app/screens/import/s3b/
├─ S3bMapping.tsx          # container: reads {sessionId, snapshot} from ImportSessionContext (2.7 seam);
│                          #   owns view state (default|config|block|worker); maps engine ↔ components
├─ use-s3b-session.ts      # hook over the EngineClient: apply/reset/confirmRecall/resolveCollision/next;
│                          #   holds the latest Stage2SnapshotDTO; derives unknown/guessed/confirmed counts
├─ RawMappingTable.tsx     # the transient sample table from snapshot.sampleCells (NO filter toolbar — decision #6);
│                          #   per-cell empty/error/ignore rendering from the cell message keys
├─ ColHeader.tsx           # per-column header: unknown ? / recalled ◇ / confirmed ✓ / ignored states (bundle)
├─ ColMenu.tsx             # instant dropdown: pick type (TYPE_ORDER) = instant apply; «More»; confirm/reconfigure/undo
├─ ConfigWizard.tsx        # the «More» 2-step: type picker → params + the scrollable markdown help panel
├─ MarkdownHelp.tsx        # the bundle's lightweight md renderer (h3/h4/table/quote/list/callout/inline) — ported
├─ StatusPanel.tsx         # default right pane: progress bar, legend, per-column status list, recall note, info intro
├─ BlockPanel.tsx          # loud UNKNOWN gate banner (Option A): names unmapped columns + jump-to-fix
├─ RejectionPanel.tsx      # >30% ColumnTransformRejection: ЩО/ЧОМУ/ДІЯ + ALL cellErrors (loud), column→UNKNOWN
└─ WorkerProgressPanel.tsx # importNext takeover (large files): 2.7's honest determinate primitive
```

- **Help-doc embedding:** the 30 `column-type-help/*.md` files are embedded at BUILD time as a typed `Record<type, {uk,en}>` map (Vite `?raw` imports or a generated module) — offline-deterministic, no fetch, no precache complexity. `info.{uk,en}.md` backs the step-intro. The bundle's `s3b-help-docs.jsx` loader stub is replaced by this map. (14 type docs + info; `ignore` & `unknown` need no doc — the dropdown labels are self-explanatory; the ConfigWizard already renders a graceful "no help yet" for any missing key.)
- **Type ↔ ColumnDefinition mapping:** the bundle's internal keys (`account`→`bank_account`, `mcc`→`merchant_category`, `rate`→`exchange_rate`, `commission`→`bank_commission`) map to the engine's ColumnDefinition strings. The 2.7 `columnTypeLabel(definition, lang)` already covers all 15 definition strings — reuse it. `TYPE_ORDER` (15 incl. ignore) drives the menu.
- **Param schema ↔ engine params:** the bundle's `PARAM_SCHEMA` maps 1:1 to the engine's accepted params — DATE `{format: 'auto'|{custom}}`, AMOUNT `{currency: auto|use_base|{code}, type: income|outcome|mixed|auto}`, BALANCE/COMMISSION/CASHBACK `{currency}`, STATUS `{successValue}`; everything else no params. `importApplyColumn(sessionId, columnId, definition, params)` takes exactly these. Verify each parseAs* accepts the param shape; expose only what the engine consumes.
- **i18n:** the bundle's `M` (uk/en chrome catalog) ports to the app catalog (1.4 pattern); reuse 2.7's LangProvider + columnTypeLabel.
- **Session reuse:** S3b reads `{sessionId, snapshot}` from the 2.7 `ImportSessionContext`. Every apply/reset/confirmRecall returns a fresh `Stage2SnapshotDTO` → update local state. «Назад» → S3a (step 1), NOT Dashboard (the wizard's internal back).

## 3. Gate #2 (canAdvance) + flows

- **`canAdvance()` #2 = zero true-UNKNOWN** (Option A): «Далі» always active; press with ≥1 UNKNOWN → BlockPanel (loud), names the columns, jump-to-first; no advance. Recalled (guessed) and confirmed and IGNORE and TIME all PASS. Advance (zero-UNKNOWN) → `importNext` → (large file: WorkerProgressPanel) → flush staged recall → S3c.
- **Instant apply:** header dropdown → pick type → `importApplyColumn` with that type's default params → snapshot refresh (confirmed state).
- **«More» 2-step:** type picker → params + help panel → Apply → `importApplyColumn` with chosen params.
- **Re-click a mapped column:** «Відмінити» (`importResetColumn` → UNKNOWN, unstage) / «Налаштувати» (reopen ConfigWizard at step 2) / «Підтвердити» on a recalled column (`importConfirmRecall` → clears the recalled flag).
- **Collision:** snapshot's `lastSaveCollision` non-null → surface the per-column param confirm (`importResolveCollision(confirm)`); type stays stable.
- **>30% rejection:** `importApplyColumn` returns a `ColumnRejectionDTO` → RejectionPanel renders ЩО/ЧОМУ/ДІЯ + ALL cellErrors; the column stays UNKNOWN; the session survives.
