## Goal

Port the prior-art crown jewel — `stage2/column.ts` (locale number parse, auto date-format detect,
amount income/outcome/mixed semantics, per-column error machinery) + the `ColumnDefinition` catalog —
and extend it with TIME + COUNTERPARTY so the catalog ≡ ENT-009 literally (16 entries). Wire the
ENT-011 currency hook to the 1.6 resolver. Engine-internal; barrel/EngineClient untouched.

## Locked decisions

1. **Luxon = engine-internal runtime dep, lazy at the parse boundary** (SheetJS precedent): dynamic
   `import('luxon')` at the transform boundary (API already async); chunk Workbox-precached; boot
   clean. Guardrails: (a) luxon types NEVER on the public surface; (b) **HC-9 pinning** — every
   `DateTime` call pins `{ zone: 'utc', locale: 'en-US' }`; a TZ-environment test parses fixed input
   under a different `TZ` and asserts identical output.
2. **Lean scope:** port `types.ts` (catalog+params), `stage2/column.ts`, `stage2/row.ts`,
   `stage2/types.ts`, + the ~400 utils lines they need (+`column.spec`/`row.spec`).
   `implementation.ts` (RxJS stage wiring) + its 1,347-line stage-interface spec suite **port INTACT
   at 2.3** (tagged there — not orphaned). 2.2's column/row-level tests stand alone.
3. **`sampleArray` determinism — MANDATORY replacement (founder upgrade):** verified 2026-06-11 —
   **the prior art uses `Math.random()`** (partial Fisher-Yates) in `collections/sampling.ts`.
   Replaced with **deterministic evenly-spaced sampling** (same signature; indices
   `floor(i·len/size)`, order-preserving) + a pinned test. Rationale: HC-9, snapshot stability, and
   the TZ-env test all presuppose it; run-to-run-varying format detection would be a silent
   wrongness generator. (Side benefit: even spacing samples the file's full date range — better
   detection across month boundaries than a random clump.)

## 2. The five adaptations (everything else verbatim, hunk-audited like 1.2/1.6)

1. `@abc-budget/logging` → `internal/logging.ts` shim (same `getLogger` surface).
2. `CurrencyCache` → **the 1.6 hook (ENT-011):** AMOUNT params carry our `CurrencyDetectOptions`;
   CURRENCY-column + AMOUNT currency resolution route through `resolveCurrency`/`symbolToIso`
   (`«грн»→UAH` works in typed columns).
3. `getEngineConfig` → static ENT-016 defaults module (constants in 2.2; settings store at 2.4).
4. Luxon lazy + pinned `{zone:'utc', locale:'en-US'}` + the TZ-environment test (decision 1).
5. `@abc-budget/utils` → ported files (not a dep), with the sampling replacement (decision 3).

## 3. The two new types (the ENT-009 gap)

- **TIME** — deliberate label-type (privacy): the transform validates nothing and OUTPUTS NOTHING.
  Test contract (the QA provoke target): typed output rows contain **no time field**, and raw time
  values appear **nowhere** in any transform output (verifiable discard). Recognized-and-IGNORED ≠
  generic IGNORE — it exists so the user labels the column explicitly (ENT-009 rationale).
- **COUNTERPARTY** — text transform like DESCRIPTION but writing the **distinct `counterparty`
  output field** (ENT-006: who vs what-for → precise rules; hashed like description in footprints,
  EP-3). Test pins it flowing separately from `description` end-to-end through row typing.

## 4. Inherited 2.1 decisions (resolved + documented)

- **xlsx date-serial policy:** under 2.1's `raw:false`, SheetJS always delivers *formatted strings* —
  raw serials cannot reach the catalog. A bare numeric string (e.g. `"45307"`) in a DATE column is a
  **per-cell parse error (loud)** — never epoch-converted (no 1900/1904 guessing; deterministic,
  honest). Documented in the catalog README + pinned by a test.
- **`col_N` placeholder names:** ordinary columnNames, no special-casing — a test types a `col_2`
  column like any named one (the 2.3 recall pool will key on them).
