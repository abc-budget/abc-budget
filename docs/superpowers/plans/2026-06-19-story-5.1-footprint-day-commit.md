# Story 5.1 — Footprint `day` + the commit pipeline (EP-5) — IMPLEMENTATION PLAN

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or
> superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax.

**Goal:** Wire the commit pipeline (EP-5, Story 5.1): a `importCommit` wire-method that categorizes the session
rows, derives 7-field footprints (now with `day`), and writes them in the EP-3 two-phase atomic batch — with a
perf-map so distinct-date rates are read once, not per-row.

**Architecture:** Assemble built pieces. Extend `deriveFootprint`/`FootprintRecord` with a UTC `day`
(derived via a new `footprintYearMonthDay` that EXTENDS the single shared `footprintYearMonth` — the 4.4.1
single-derivation invariant); `day` is a plain DATA field (not in the `[hash,year,month]` key → no IDB
migration). Extend the EP-3 `commitFootprints` two-phase with a preloaded `Map<dateKey, ratesTable>` (read once)
+ an optional per-row `categoryOf`. A worker `commitSession` runs `autoCategorize` → `commitFootprints` → frees
the session. `importCommit` is the v7 wire method (CONTRACT_VERSION 6→7); it frees the session ONLY on success
(a `RatesUnavailableError` propagates first — the loud gate, zero writes, session retained for retry).

**Tech Stack:** TypeScript; `packages/engine`; vitest + fake-indexeddb; the versioned Web-Worker EngineClient wire.

## Global Constraints

- **`CONTRACT_VERSION` 6 → 7** (per-story bump; same handshake → `ContractMismatchError`/loud-reload). A v7 UI
  must reject a stale v6 worker.
- **The footprint is EXACTLY 7 fields** after this story: `{ year, month, day, amountUSD, categoryId, isManual,
  hash }`. The compile-pin (`footprint/types.spec.ts`) goes 6→7. `day` is privacy-neutral (a UTC day number).
  `amountUSD` stays (ENT-020, INTERNAL — never read by any UI). No secrets in the record.
- **`day` is a plain DATA field** — the key stays `['hash','year','month']`; `day` is NOT in the key or any
  index → **NO IDB migration**.
- **Single-derivation invariant (4.4.1):** a row and its footprint compute the SAME y/m/d. `footprintYearMonthDay`
  EXTENDS `footprintYearMonth` (the override key still uses the y/m-only `footprintYearMonth` — `day` is not in
  the key). UTC accessors only (`getUTCFullYear`/`getUTCMonth`+1/`getUTCDate`).
- **Two-phase ATOMIC commit:** build ALL records (convert + derive) in a pre-flight pass; call `putBatch` ONLY
  if every convert succeeded. One uncached rate among N rows → `RatesUnavailableError` propagates BEFORE any
  write → ZERO rows persisted (not partial). The cache-only read is the single loud gate.
- **Perf-map (EP-3 carry-forward):** preload the distinct operation-date rates into a `Map` ONCE (M reads, not
  N sequential per-row IDB reads); the per-row convert reads the preloaded map. Reuse `toAmountUSD` (direction-
  correctness + the loud gate preserved) by feeding it a cache-backed reader.
- **Dump'd rows commit `isManual=0`** (DERIVED, not manual) → on re-import they are uncategorized again
  (the 4.6 transient guarantee). Manual overrides (L1/L2) commit `isManual=1` → preserved on re-import. Re-import
  upserts by `(hash, year, month)` (native IDB last-write-wins).
- **Free the session after a SUCCESSFUL commit** (raw not retained). On `RatesUnavailableError` the session is
  NOT freed (retry-able). The free reuses the `importAbort` path (`registry.free` + the `dropSandbox`/`dropDump`
  teardown).
- **`CommitResultDTO` is type-only** (barrel) — not a runtime symbol; the barrel stays 3 runtime symbols.
- **REAL-DTO multi-rule / multi-currency fixtures** (the recurring teeth) — the categorized commit (footprint
  categoryId/isManual per row) + the magnitude/convert + the loud gate are the prime false-green spots.

```ts
export interface CommitFootprintsDeps {
  footprintDao: FootprintDao;
  ratesDao: ExchangeRateDAO;
  warm: (dates: Date[]) => Promise<void>;
  /** 5.1: per-row categorization. Defaults to derived (null/0) — the EP-3 raw behavior. */
  categoryOf?: (row: TransactionRow) => { categoryId: string | null; isManual: 0 | 1 };
}

export async function commitFootprints(rows: readonly TransactionRow[], deps: CommitFootprintsDeps): Promise<CommitResult> {
  const distinct = distinctOpDates(rows);
  try { await deps.warm(distinct); } catch { /* best-effort; the cache-only convert is the loud gate */ }

  // PERF-MAP: preload the distinct-date USD rate tables ONCE (M reads, not N).
  const byDate = new Map<string, ExchangeRateEntity | null>();
  for (const date of distinct) {
    const key = date.toISOString().split('T')[0];
    if (!byDate.has(key)) byDate.set(key, await deps.ratesDao.findByBaseCurrencyAndDate('USD', key));
  }
  const ratesReader = new PreloadedRatesDao(byDate);

  // PRE-FLIGHT (PURE): convert + derive EVERY row from the preloaded map. A cache
  // miss throws RatesUnavailableError here — before any write (the two-phase guarantee).
  const records: FootprintRecord[] = [];
  for (const row of rows) {
    const amountUSD = await toAmountUSD(row.amount, row.currency, row.date, ratesReader);
    const cat = deps.categoryOf ? deps.categoryOf(row) : { categoryId: null, isManual: 0 as const };
    records.push(deriveFootprint(row, amountUSD, cat.categoryId, cat.isManual));
  }

  await deps.footprintDao.putBatch(records);
  return { written: records.length };
}
```
