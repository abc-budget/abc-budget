## Task 1 — Canonical condition-equality  [Wave A]
**Files:** `rules/canonical-conditions.ts` (+spec).
- `export function canonicalConditionSignature(rules: Rule[]): string` — per condition a canonical key
  `${field}|${operation.type}|${stableSerialize(operation)}`; **SORT the keys**, join with a separator. Two
  condition sets are equal iff signatures match — **order-INDEPENDENT** (AND-combined ⇒ order non-semantic).
  `stableSerialize(operation)` produces a key-order-stable string (sort object keys) so `{type,value}` ≡
  `{value,type}`.
- `export function conditionsEqual(a: Rule[], b: Rule[]): boolean` = signatures match.
- Doc-comment: **why this is separate from 4.3b `rulesContentMatch`** — that matcher is POSITIONAL
  (`a.rules[i].field !== b.rules[i].field` by index, for store-delta id reuse); it would falsely flag a
  condition REORDER as "changed". The no-op detector needs reorder-invariant equality. Keep both.
- **Tests (teeth):** same conditions in DIFFERENT order → `conditionsEqual === true` (the no-op);
  a changed value/operator/field → `false`; a different count → `false`; the signature is stable across runs
  (determinism); a `matches` RegExp operation serializes stably.

## Task 2 — `RuleSandboxSession`  [Wave B — needs T1]
**Files:** `rules/rule-sandbox.ts` (+spec).
- `type EditAction = { kind:'reorder'; order:number[] } | { kind:'delete'; ruleId:number } |
  { kind:'editConditions'; ruleId:number; before:Rule[]; after:Rule[] } | { kind:'categoryOnly'; ruleId:number;
  category:Category } | { kind:'appendEnd'; rule:ComplexRule }`.
- `class RuleSandboxSession`, ctor `({ importRows, overrideMap, categoriesById, currentTree, persistence })`.
  State: `currentTree`, `virtualTree: DecisionTree|null`, `get engaged()`.
- **`classify(action): 'sandbox' | 'live'` — PURE SYNC:** reorder→sandbox; delete→sandbox; categoryOnly→live;
  appendEnd→live; editConditions→`conditionsEqual(before, after) ? 'live' : 'sandbox'` (T1). O(1)/bounded,
  NEVER consults the diff. This IS the dynamic-label decision (live="update" / sandbox="review").
- **`computeDiff(): { rowIndex; oldCategoryId: string|null; newCategoryId: string|null }[]` — PURE SYNC:** for
  each importRow, `oldC = resolveCategory(row, { overrideMap, categoriesById, tree: currentTree })`,
  `newC = resolveCategory(row, { overrideMap, categoriesById, tree: virtualTree ?? currentTree })`; emit when
  `(oldC?.id ?? null) !== (newC?.id ?? null)`. Empty when not engaged. **ZERO DB** (maps in-memory, evaluator sync).
- **`async submit(action): Promise<'sandbox'|'live'>` — the funnel:** `const d = classify(action)`;
  - if `d==='live' && !engaged`: apply the action to `currentTree` (via the builders) AND **persist immediately**
    (categoryOnly → `persistence.update(updatedRule)`; appendEnd → `persistence.create(newRule)`); set
    `currentTree` to the rebuilt tree.
  - else (`d==='sandbox'` OR already `engaged`): ENGAGE if needed (`virtualTree ??= currentTree`); ACCUMULATE —
    `virtualTree = applyActionToTree(virtualTree, action)` (rebuild via builders). NO persist yet.
  - return `d`.
- **`async apply(): Promise<void>`:** require engaged; `await persistence.saveDecisionTree(virtualTree)` (4.3b
  delta → matched rules keep ids); `currentTree = virtualTree`; `virtualTree = null`.
- **`cancel(): void`:** `virtualTree = null` (discard; current unchanged). Sync (no persist).
- **`applyActionToTree(tree, action): DecisionTree`** — a pure helper using `DecisionTreeBuilder`/
  `ComplexRuleBuilder`: reorder → complexRules reordered per `order`; delete → drop `ruleId`; editConditions →
  replace that rule's `rules`; categoryOnly → replace its `category`; appendEnd → push the new complexRule.
  Always rebuild via the builders (never hand-mutate).
- **Tests (the 5 proofs):**
  1. **trigger table** — each action kind → the expected sandbox|live, O(1), and INDEPENDENT of the diff
     (a delete that changes 0 rows still classifies `sandbox`).
  2. **diff correct + ZERO DB** — engage a sandbox edit; `computeDiff()` returns exactly the rows whose
     resolved category changed (old→new); spy the footprint + complexRule DAOs → ZERO reads across repeated
     `computeDiff()`/`classify()` calls.
  3. **apply preserves IDs / cancel reverts** — build current with rules having ids; sandbox a reorder+edit;
     `apply()` → `saveDecisionTree` called, matched rule ids preserved (assert via the persistence mock/real),
     `currentTree` becomes virtual; a fresh session: `cancel()` → `currentTree` unchanged, `engaged===false`.
  4. **same-conditions-any-order = no sandbox** — `submit({editConditions, before:[A,B], after:[B,A]})` →
     `classify==='live'`, session does NOT engage (`engaged===false`).
  5. **override-ops NEVER in the diff** — an importRow with an override (L1 flag OR L2 map hash) that a rule
     ALSO matches; sandbox a delete of that rule → `computeDiff()` does NOT include that row (it resolved via
     L1/L2 under both trees). The override-independence proof.
- DI a mock or real `RulePersistenceService` for the persist assertions; build trees via the builders;
  `resolveCategory` is the 4.4 import.
