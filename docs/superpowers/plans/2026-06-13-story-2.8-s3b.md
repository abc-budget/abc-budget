## Task 1: Defer-commit recall — staged writes, flush-on-advance (engine, TDD)

**Files:** MODIFY `internal/importStatement/stage2/implementation.ts` (staging buffer + applyColumn detect-not-write + flush/unstage methods) · MODIFY `internal/importStatement/recall/recall.ts` (a read-only `detectCollision` if not already separable from `save`) · MODIFY `client/direct-client.ts` (importNext flush; importAbort discard; importResetColumn unstage) · ADAPT `implementation.recall.spec.ts` + any recall suite asserting apply-time writes (declared callouts) · the worker-host inherits via the direct-client shim (parity).

- [ ] **Buffer keyed by columnId** (item 1 resolution): `_stagedRecallWrites: Map<string /*columnId*/, { name: string; definition: ColumnDefinition; params: ColumnParams | null }>`. Keying by id (not normalizedName) makes unstage per-column-correct and defers name-collapse to flush where it matches the pool's LWW.
- [ ] **`applyColumn` change** (failing tests first): when `_recallPool && column.definition !== null`, replace the immediate `_recallPool.save(...)` write with: (1) read-only collision DETECT (`getEntry(normalizeKey(name))` + the existing type-change/params-change comparison) → set/clear `lastSaveCollision` EXACTLY as today (map-time UX byte-identical — assert lastSaveCollision parity); (2) stage `{ name, definition, params }` under `column.id`. NO IDB write. Keep non-blocking + loud-on-failure (HC-7) for the async detect.
- [ ] **`flushRecallWrites(): Promise<void>`** (new): for each staged entry, commit preserving today's semantics — new/identical → `recallPool.save()` (writes); a still-unresolved params-change → `save()` returns collision and does NOT write (keeps the stored entry — the safe no-clobber default, item 3); an entry the user resolved-confirm → `recallPool.confirmSave()` (LWW). Track per-staged-entry whether the user confirmed (set by importResolveCollision). Await all writes; log loud on failure (HC-7).
- [ ] **`unstageRecallWrite(columnId): void`** (new): `_stagedRecallWrites.delete(columnId)`. Clean per-id; a sibling column sharing a normalized name is untouched.
- [ ] **direct-client wiring:** `importNext` → `await flushRecallWrites()` (before freeing the session; alongside `generateForSession`); `importAbort` → free WITHOUT flush (buffer dies with the stage2 instance = discard); `importResetColumn` → `unstageRecallWrite(columnId)` after the reset. `importResolveCollision(confirm)` → mark the staged entry confirmed/keep (so flush picks the right write path) in addition to the existing confirmSave-on-resolve.
- [ ] **Rejection ordering** (pin e): the >30% `ColumnTransformRejection` throws inside the transform BEFORE staging — a rejected column never stages, stays UNKNOWN. Assert.
- [ ] **Pins (a)–(g) — each an explicit test** (real-hop where it proves parity): (a) apply→abort→fresh importStart ⇒ pool empty / no recall; (b) apply→advance(importNext)→fresh importStart ⇒ recalled (guessed); (c) collision fires at apply-time on a staged map (lastSaveCollision set, no write yet); (d) importResetColumn discards a staged map (apply→reset→advance ⇒ not written); (e) >30% rejection at apply, column→UNKNOWN, not staged; (f) HC-9 determinism (same session twice → identical pool state); (g) Direct ≡ Worker parity (the round-trip over the real worker hop equals the direct client).
- [ ] **Item 1 test (dup/blank/NFC-collapse):** two columns whose `originalName`s collapse to one `normalizeKey` (NFC vs NFD encoding of the same header, constructed via a decode or crafted stage2) → both stage under distinct ids → flush → pool has ONE entry (last-applied wins = 2.3 LWW, NO new clobber); unstage one → the other's staged write survives to flush. Plus: the `empty-header-cells.csv` fixture's `col_N` placeholder maps + stages like any other name.
- [ ] Gate + commit: `feat(2.8): defer-commit recall — staged writes keyed by columnId, flush on advance, discard on abort (decision #4 pins a–g)`.
