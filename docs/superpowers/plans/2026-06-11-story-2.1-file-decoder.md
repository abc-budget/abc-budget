# Story 2.1 — file→rows decoder (THE WEDGE) — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** `decode(input) → { rows, issues, meta }` — any messy bank statement in, raw rows + a complete coordinate-addressed issue list out, never an unstructured exception, deterministic; synthetic fixture matrix + hard-count local real-statement spec; QA harness page proving offline .xls decode.

**Architecture:** One messiness pipeline, two front-ends: CSV bytes → `encoding.ts` → `csv-parser.ts` → matrix; XLS/XLSX → lazy SheetJS → matrix; both → `header-detect.ts` → keyed rows. Every module collects `DecodeIssue`s (ЩО/ЧОМУ/ДІЯ + coordinates) instead of throwing. The engine exports an UNSTABLE `"./qa"` subpath (decode only) consumed by `qa-harness.html` (third Vite entry) — sunset at 2.6.

**Tech Stack:** SheetJS CE `xlsx@0.20.3` via pinned official CDN tarball (lazy `import()`), own CSV parser (zero deps), TextDecoder('windows-1251') for cp1251, vitest snapshots.

## Task 1: types + encoding (TDD)

- [ ] **Failing tests `encoding.spec.ts`** (table-driven; build byte arrays inline):
  - UTF-8 plain ASCII → `{ text, encoding: 'utf-8', bom: false }`.
  - UTF-8 with BOM (`EF BB BF` prefix) → BOM stripped, `bom: true`.
  - Valid multi-byte UTF-8 («Дата» as UTF-8 bytes `D0 94 D0 B0 D1 82 D0 B0`) → utf-8.
  - cp1251 «Дата» (`C4 E0 F2 E0`) — INVALID as UTF-8 → decoded via windows-1251, `encoding: 'windows-1251'`.
  - cp1251 full Ukrainian header line (bytes for «Дата операції;Сума;Опис») → round-trips to the expected string.
  - Ambiguous pure-ASCII → utf-8 (no issue); bytes invalid in BOTH (e.g. lone `0xC4` at EOF) → still returns text (windows-1251 maps every byte) + caller-visible `ambiguous: true` flag.
- [ ] **Implement `types.ts`** — exactly the spec §1 contract:

```ts
export interface DecodeInput { bytes: ArrayBuffer; fileName: string }
export type DecodeAction =
  | 'skipped-row' | 'kept-raw' | 'padded-row' | 'truncated-row'
  | 'recovered-quote' | 'renamed-column' | 'file-unreadable' | 'no-data';
export interface DecodeIssue {
  row: number;            // 0-based row in the SOURCE file; -1 = file-level
  column?: string | number;
  what: string;           // ЩО happened (short, key-like English; UI localizes at 2.8)
  why: string;            // ЧОМУ — human reason with specifics
  raw?: string;           // the offending raw value, truncated to 200 chars
  action: DecodeAction;   // ДІЯ taken
}
export interface DecodeMeta {
  format: 'csv' | 'xlsx' | 'xls';
  encoding?: 'utf-8' | 'windows-1251';
  bom?: boolean;
  delimiter?: ',' | ';' | '\t';
  headerRow: number;      // 0-based source row; -1 if none found
  sheet?: string;
  otherSheets?: string[];
  totalRows: number;      // physical rows seen (excl. header) in source
  decodedRows: number;    // rows emitted
}
export interface DecodeResult {
  rows: Record<string, unknown>[];
  issues: DecodeIssue[];
  meta: DecodeMeta;
}
```

- [ ] **Implement `encoding.ts`**: `decodeBytes(bytes: ArrayBuffer): { text: string; encoding: 'utf-8'|'windows-1251'; bom: boolean; ambiguous: boolean }` — BOM check; strict pass with `new TextDecoder('utf-8', { fatal: true })` in try/catch; on failure decode with `new TextDecoder('windows-1251')`; `ambiguous` true only when the cp1251 path produces <1% Cyrillic among letters (heuristic sanity score — pure function, unit-tested).
- [ ] Gate + commit: `feat(2.1): ingest types + encoding detection (BOM/UTF-8/cp1251)`.

## Task 2: tolerant CSV parser (TDD)

- [ ] **Failing tests `csv-parser.spec.ts`** — the contract, as code:

```ts
// sniffDelimiter
expect(sniffDelimiter(['a;b;c', '1;2;3'])).toBe(';');
expect(sniffDelimiter(['"a,x",b', '1,2'])).toBe(',');          // quoted commas don't count
expect(sniffDelimiter(['a\tb', '1\t2'])).toBe('\t');
expect(sniffDelimiter(['1;2,5;3', '4;5,1;6'])).toBe(';');      // comma-decimal trap: ; wins by consistency
// parseCsv — happy
expect(parseCsv('a,b\n1,2', ',').matrix).toEqual([['a','b'],['1','2']]);
// quoted field with embedded delimiter + escaped quote ("" → ")
expect(parseCsv('"x,y","he said ""hi"""', ',').matrix[0]).toEqual(['x,y', 'he said "hi"']);
// RECOVERY: bare quote inside an unquoted field — kept as literal, issue recorded
const r1 = parseCsv('plain "oops field,next\nrow2,b', ',');
expect(r1.matrix[0][0]).toBe('plain "oops field');
expect(r1.issues[0]).toMatchObject({ row: 0, action: 'recovered-quote' });
// RECOVERY: unterminated quoted field — closed at newline, issue recorded
const r2 = parseCsv('"never closed,a\nnext,b', ',');
expect(r2.matrix).toHaveLength(2);
expect(r2.issues[0].action).toBe('recovered-quote');
// CRLF + trailing newline normalization
expect(parseCsv('a,b\r\n1,2\r\n', ',').matrix).toHaveLength(2);
// empty lines skipped with issue
const r3 = parseCsv('a,b\n\n1,2', ',');
expect(r3.matrix).toHaveLength(2);
expect(r3.issues.some(i => i.action === 'skipped-row')).toBe(true);
```

- [ ] **Implement** `sniffDelimiter(lines: string[]): ','|';'|'\t'` — score each candidate over the first ≤20 non-empty lines: count occurrences OUTSIDE quotes per line; a candidate scores by (median count > 0) + low variance (consistency); ties → `;` > `\t` > `,` (comma last — comma-decimal trap). `parseCsv(text, delimiter): { matrix: string[][]; issues: DecodeIssue[] }` — single-pass char state machine (states: FIELD_START / IN_FIELD / IN_QUOTES / QUOTE_IN_QUOTES); recovery rules exactly as the tests pin: bare quote mid-field → literal + issue; unterminated quote → close at line end + issue; `""` inside quotes → escaped quote. Track source row numbers through skips (issues carry SOURCE coordinates).
- [ ] Gate + commit: `feat(2.1): tolerant CSV parser — delimiter sniffing + quote recovery`.

## Task 3: header detection + row keying (TDD)

- [ ] **Failing tests `header-detect.spec.ts`** — contract:

```ts
const PRIVAT_LIKE = [
  ['Виписка з картки', '', ''], ['Клієнт: ТЕСТ ТЕСТОВИЧ', '', ''], ['', '', ''],
  ['Дата', 'Опис', 'Сума'],
  ['01.10.2023', 'Покупка', '-120,50'], ['02.10.2023', 'Кафе', '-89,00'],
  ['Разом', '', '-209,50'],
];
const d = detectHeader(PRIVAT_LIKE);
expect(d.headerRow).toBe(3);
expect(d.keys).toEqual(['Дата', 'Опис', 'Сума']);
// preamble rows produce skipped-row issues with coordinates 0..2
expect(d.issues.filter(i => i.action === 'skipped-row').map(i => i.row)).toEqual([0, 1, 2]);

// placeholder row (the real mono « - » case) and trailing summary are data-phase concerns:
const rows = keyRows(PRIVAT_LIKE, d);
expect(rows.rows).toHaveLength(2);                                  // 2 data rows
expect(rows.issues.some(i => i.row === 6 && i.action === 'skipped-row')).toBe(true); // «Разом» summary
// placeholder row of " - " cells:
const withPlaceholder = [['a','b'],[' - ',' - '],['1','2']];
const k = keyRows(withPlaceholder, { headerRow: 0, keys: ['a','b'], issues: [] });
expect(k.rows).toEqual([{ a: '1', b: '2' }]);
expect(k.issues[0]).toMatchObject({ row: 1, action: 'skipped-row' });
// duplicate header names → deterministic suffix + issue
const dup = detectHeader([['Сума', 'Сума', 'Дата'], ['1', '2', '01.01.2024']]);
expect(dup.keys).toEqual(['Сума', 'Сума_2', 'Дата']);
expect(dup.issues.some(i => i.action === 'renamed-column')).toBe(true);
// ragged data row → padded + issue; longer → extra positional key + issue
// no detectable header → headerRow -1, positional keys col_1.., file-level issue
```

- [ ] **Implement** `detectHeader(matrix): { headerRow: number; keys: string[]; issues: DecodeIssue[] }` — scan the first ≤25 rows; score = (non-empty distinct string cells / width) × (next-3-rows width-consistency) × (non-numeric-majority bonus); best score above floor wins; below floor → headerRow -1 + positional keys. `keyRows(matrix, header): { rows; issues }` — skip pre-header rows (issue each), skip all-empty / all-placeholder (cells ∈ {'', '-', '—', 'N/A'} after trim) rows (issue), detect trailing summary rows (≤40% filled AND first cell matches /^(Разом|Всього|Total|Итого)/i — both conditions) → skip + issue; pad/truncate ragged rows (+issue, `padded-row`/`truncated-row`); dedup keys with `_2`, `_3` suffixes (+`renamed-column` issue).
- [ ] Gate + commit: `feat(2.1): header detection — preamble skip, junk/summary rows, key dedup`.
