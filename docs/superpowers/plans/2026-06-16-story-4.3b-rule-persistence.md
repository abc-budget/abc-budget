## Task 3 — `complex-rules-dao.ts` (DTO + (de)serialize + DAO)  [Wave B — needs T1 + T2]
**Files:** `internal/rules/complex-rules-dao.ts` (+ spec). Template: prior-art `decision-tree/dao.ts` +
`internal/importStatement/recall/pool-dao.ts` (IDBBatchDao pattern). Read `store/idb/dao-impl.ts` for the
`IDBBatchDao` API (create/read/update/list/delete/batch*).
- `RuleDTO { field: string; operation: RuleOperation }`; `ComplexRuleDTO { id?: number; rules: RuleDTO[];
  categoryId: string; order: number }`.
- `serializeRule(rule)` → `{field, operation}`; `deserializeRule(dto)` → `rehydrateRule(dto.field,
  dto.operation)` (T2). `serializeComplexRule(complexRule, order)` → DTO (throws `LocalizableException` via
  `createLocalizableMessage('engine.rules.complex-rule.category-id-required')` if `category.id` missing);
  `categoryId = category.id` (STRING).
- `ComplexRuleDAO extends IDBBatchDao<number, ComplexRuleDTO>` on `'complexRules'`, keyPath `'id'`. CRUD +
  `getAllOrdered()` (sort by `order`) + `getByCategoryId(categoryId: string)` (via index). Direct DI.
- **Pins:** create (no id) → DAO assigns an autoInc number; round-trip a DTO; `getAllOrdered` sorts;
  `getByCategoryId` filters; serialize throws on a category with no id; `deserializeRule` rebuilds an
  evaluating Rule (delegates to T2's covered path).

## Task 4 — `rule-persistence-service.ts` (RxJS-FREE)  [Wave C — needs T3]
**Files:** `internal/rules/rule-persistence-service.ts` (+ spec). Constructor `(complexRuleDao,
categoriesService)` — direct DI (the categories SERVICE so «base» resolves at read).
- `create(complexRule)` → order = max(existing.order)+1 (0 if none); `dao.create(serialize(rule, order))`.
- `update(complexRule)` → preserve the existing DTO's `order`; `dao.update`.
- `reorder(ruleIds: number[])` → read `id→order` from `getAllOrdered`; `reorderRuleOrders(currentOrders,
  ruleIds)` (4.2); write each changed DTO's new `order` (batch). **Only `order` changes; ids preserved.**
- **`reload(): Promise<DecisionTree>` — SYNCHRONOUS, no RxJS:** `getAllOrdered()` → for each DTO, rehydrate
  `rules` (deserializeRule) + resolve `category = await categoriesService.get(dto.categoryId)`. **Dangling
  (`get` → null): SKIP the rule + `logger.error` (RULING 1) + leave the DTO in the store.** Build a fresh
  `DecisionTreeImpl` with the surviving `complexRules` IN `order` SEQUENCE (array order = eval order).
- **`saveDecisionTree(tree)` — delta-ID-preservation (4.5's Apply path):** for each complexRule with no
  `category.id`, `categoriesService.create(category)` to mint one; list existing DTOs; content-match each
  tree rule to an unused existing DTO (same `categoryId` + same serialized `rules`) → **preserve its id**;
  unmatched tree rules → create (fresh autoInc id); existing DTOs not matched → **delete**. Order = tree index.
  RxJS-free. **Policy-free** — no live-vs-sandbox decision here.
- **Pins (the 6 acceptance proofs land across T1+T4):** create→reload round-trip (rehydrated rule evaluates
  identically); reorder ids preserved + new order survives reload (first-match honors it); saveDecisionTree
  {keep A / add C / remove B / reorder} → A keeps id, C fresh id, B deleted (explicit id assertions);
  dangling categoryId → rule skipped + logged + DTO retained; a «base»-currency category resolves through
  4.3a at reload; `grep rxjs|BehaviorSubject|Observable` in the new files = 0.
