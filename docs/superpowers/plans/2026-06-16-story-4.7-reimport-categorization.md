## The assembly — `rules/auto-categorize.ts`
```ts
interface CategorizedRow { row: ImportStatementStage3Row; categoryId: string | null; isManual: 0 | 1; }
interface AutoCategorizeDeps {
  tree: DecisionTree;
  footprintDao: FootprintDao;
  categoriesService: CategoriesService;
  dumpCategoryId?: string | null;   // 4.6 L4 dump (default null)
}
async function autoCategorize(rows: readonly ImportStatementStage3Row[], deps): Promise<CategorizedRow[]>
```
Steps:
1. **periods** = distinct `{year, month}` from each `row.date` via **UTC** accessors (`getUTCFullYear`,
   `getUTCMonth()+1`) — consistent with `deriveFootprint`/the footprint store key.
2. **`loadOverrideMap(footprintDao, categoriesService, periods)` — exactly ONCE** (4.4; period-scoped, the
   only DB touch). → `{ overrideMap, categoriesById }`.
3. `ctx = { tree, overrideMap, categoriesById }`. Per row (pure, sync):
   - `categoryId = effectiveCategory(row, ctx, dumpCategoryId ?? null)?.id ?? null` (4.6 — the full
     L1→L2→L3→L4 ladder).
   - `isManual = (row.isManuallySetCategory || overrideMap.has(row.hash)) ? 1 : 0` — the manual SOURCE flag
     (L1 in-session pick OR L2 persisted override). L3 rule / L4 dump → 0. **Decision:** the batch emits
     `isManual` (not just `categoryId`) because the Q-007 contract is "categoryId + isManual" and EP-5's
     `deriveFootprint(row, usd, categoryId, isManual)` consumes both. The footprint WRITE stays EP-5.
4. Return the `CategorizedRow[]`. **Deterministic + sync after the one-time load.**

## Acceptance proofs (tests-with-teeth) — `rules/auto-categorize.spec.ts`
- **DETERMINISTIC:** same `(rows, tree, overrideMap-source, dumpCategoryId)` → byte-identical output across two runs.
- **LOAD-ONCE / zero per-op DB:** spy the footprint DAO — `getManualByPeriods` called exactly ONCE for the
  whole batch; ZERO further reads during the per-row resolution (the maps are in-memory, the evaluator sync).
- **Q-007 RE-IMPORT CONTRACT (headline):** seed the footprint store with a mix — some rows `isManual=1`
  (manual overrides, with a categoryId), some `isManual=0` (rule-categorized). RE-IMPORT the same statement
  (same hashes) with a CHANGED tree; `autoCategorize` loads the override map from the store:
  - `isManual=0` rows → re-resolve to the NEW rule's category (updated, not frozen);
  - `isManual=1` rows → KEEP their manual category via L2 (rule change ignored), output `isManual=1`;
  - idempotent: re-import SAME rows + SAME tree → identical output (no divergence).
- **manual UNAFFECTED by rule edits:** delete/reorder a rule → the L2 (manual) row's category + isManual are unchanged.
- **NOT a full-store re-run:** the only DB read is the period-scoped `loadOverrideMap`; `autoCategorize`
  operates on the passed-in rows and NEVER scans/re-categorizes the whole footprint store (Q-007 — can't, by
  minimization). Assert the footprint DAO is queried only via `getManualByPeriods(periods)`, never a full `getAll`.
