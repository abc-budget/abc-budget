## Confirmed rulings (binding — new KB truth)
- **FORK A:** a thin SYNC orchestrator `resolveCategory(row, {overrideMap, categoriesById, tree})` ABOVE the
  bare `categorizeRow` — NOT baked into the 4.2 evaluator. Q-012 requires it: the 4.5 sandbox virtualizes
  ONLY level-3 (rules); overridden ops (L1/L2) must never appear in the rule-diff, so L1–L2 sit above the
  bare tree. Precedence: (1) `row.isManuallySetCategory && row.category` → (2) `overrideMap.get(hash)` →
  (3) `tree` first-match → (4) null.
- **FORK B (UPGRADED):** store **`isManual` as `0 | 1`** (NOT boolean — a boolean keyPath drops the tuple
  from any compound index). **Compound non-unique index `[year, month, isManual]`** on footprint. **Period-
  scoped load:** at import start, distinct `(year,month)` from the parsed rows → per period
  `index.getAll(IDBKeyRange.only([year, month, 1]))` → `Map<hash, categoryId>`. Load-ONCE, in-memory →
  `categorizeRow` stays synchronous (no per-op DB). Exact, no over-fetch.
- **CONFIRM 3:** `deriveFootprint` is the single footprint constructor; now emits 6 fields, takes the
  RESOLVED `categoryId` + `isManual` (1 ONLY when the win was L1/L2, else 0). EP-5 commit trigger OUT (4.4
  proves the round-trip via persist→load→resolve in tests).
- **CONFIRM 4:** KB cleared (Q-012/ENT-014/FEAT-020; the orphaned-override edge — op vanished/hash changed —
  is a FUTURE sanitization candidate, flag only).
- `CONTRACT_VERSION` 3; no wire method; nothing to the public barrel.

## Task 3 — Override module: load + resolve + reset  [Wave C — needs T2 + T1a]
**Files:** `rules/categorize-with-overrides.ts` (+spec). (Lives in `rules/` — the categorization layer.)
- `loadOverrideMap(footprintDao, categoriesService, periods): Promise<{ overrideMap: Map<string,string>;
  categoriesById: Map<string, Category> }>` — `getManualByPeriods(periods)` → `Map<hash, categoryId>`;
  `categoriesService` (getAll, «base» resolved) → `Map<id, Category>`. Load-ONCE.
- `resolveCategory(row, ctx: { overrideMap, categoriesById, tree }): Category | null` — SYNC:
  L1 `row.isManuallySetCategory && row.category` → row.category; L2 `const id = overrideMap.get(row.hash);
  if (id) { const c = categoriesById.get(id); if (c) return c; /* dangling → logger.error + fall through */ }`;
  L3 `tree.categorizeRow(row)`; L4 null. NO DB access (maps pre-loaded).
- `resetToRules(row, ctx)` — L1: `row.isManuallySetCategory = false; row.category = null`; L2:
  `ctx.overrideMap.delete(row.hash)`. (Next commit: `deriveFootprint` reads the cleared flag → isManual 0 +
  categoryId = rule result; the native upsert overwrites — no separate delete.)
- **Pins (the 6 acceptance proofs + the seam):**
  1. **override survives re-import** — `deriveFootprint(manualRow, usd, catId, 1)` → `footprintDao.put` →
     `loadOverrideMap([period])` → `resolveCategory(reImportedRow)` returns the manual category.
  2. **L1 > L2** — a row with BOTH the in-session flag and a map entry → the in-session category wins.
  3. **isManual=0 re-evaluates** — a non-manual row not in the map → `resolveCategory` runs rules; after a
     rule change it re-evaluates (not frozen).
  4. **loaded ONCE / no per-op** — spy/count footprint DAO reads across a categorize pass over N rows →
     only the load-once reads, ZERO per-op.
  5. **sandbox-independent** — an override-op keeps its category across a rule reorder/delete (L2 > L3).
  6. **reset** — `resetToRules` then `resolveCategory` returns the row's RULE result.
