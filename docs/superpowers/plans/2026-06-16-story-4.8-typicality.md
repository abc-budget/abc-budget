## Confirmed rulings (binding)
1. **Combiner = NOISY-OR** `1 − Π(1−a_f)` (now ENT-021 truth; the weighted-sum diluted the single strong signal).
2. **Constants are STARTING values** (tunable). `T_abs=0.6` is the volume knob → **validate on a REAL statement
   + report flag-rate** in Dev-complete; >~15–20% on a clean bucket → flag for re-tune.
3. Categorical informativeness gate **`p_mode ≥ 0.5`** (uniform-diverse killer).
4. **No signal-strength prior in the SCORE** (strength orders attribution) — EXCEPT the text containment (#6b).
5. Engine fn is **moment-AGNOSTIC** (bucket → typicality); rule-create vs (re)import wiring is 4.9.
6. **BINDING TEXT FIX:** lexical df-novelty over real descriptions over-fires (IDs / store numbers / varied
   merchant names → all "rare" → every op flagged → breaks empty-tail). Required:
   - **(a) defensive tokenization:** lowercase; keep ONLY purely-alphabetic tokens of length ≥ 3 (`/^[a-z]{3,}$/`);
     drop numeric / alphanumeric-ID / short tokens. ("stock"/"fee"/"atb" survive; "123"/"atb123"/"#"/"of" don't.)
   - **(b) text CANNOT flag alone:** `a_text = min(rawTextAtypicality, TEXT_CAP)` with **`TEXT_CAP = 0.5 < T_abs`**.
     Via noisy-OR a text-only op tops out at 0.5 < 0.6 → never crosses the tail; MCC/amount/categorical still
     flag alone; text only PUSHES an already-suspicious op further. (Targeted containment of a known-noisy
     signal — consistent with Ruling 4, not a general prior.)
   - **(c) hardened test:** a homogeneous-CATEGORY bucket with VARIED merchant descriptions → **EMPTY tail**.

## Task 1 — `typicality/profile.ts`: field profiles + per-field `a_f` (the math core)  [Wave A]
**Files:** `internal/rules/typicality/profile.ts` (+ `constants.ts`) (+spec).
- Build per-bucket profiles: categorical histograms (value→count, mode, p_mode, coverage); amount per-currency
  `{median, MAD, count}`; text BoW (token→df) over **defensively-tokenized** (6a) non-filter tokens.
- `informativeFields(profile, filteredFields)` → the set passing the gates (categorical: coverage≥0.5 ∧
  p_mode≥0.5 ∧ not a filtered categorical field; amount: per-currency count≥5; text: ∃ core token df≥0.5).
- per-field `a_f(op) ∈ [0,1]`:
  - categorical: `clamp(1 − p(op.value)/p_mode, 0, 1)` (mode→0).
  - amount: `z=|amt−median_c|/(MAD_SIGMA·MAD_c)`; `clamp((z−Z0)/(ZMAX−Z0),0,1)`; `MAD_c=0` → `amt==median?0:1`.
  - text: `min(max over op's kept tokens of (1−df(t)), TEXT_CAP)` (6b — capped below T_abs).
- **Tests:** each a_f (mode→0, minority→high, amount MAD-z ramp + MAD=0 cases, per-currency isolation, text
  rarest-token + **the cap** + **defensive tokenization** drops IDs/numbers/short); the gates (uniform-diverse
  categorical → not informative; constant field → a_f 0 for all; filtered field excluded; amount currency
  <5 → not informative for its ops).

## Task 2 — `typicality/index.ts`: `rankBucket` (assemble + tail + attribution)  [Wave B — needs T1]
**Files:** `internal/rules/typicality/index.ts` (+spec).
- `rankBucket(rows, filteredFields)`: N-gate (`rows.length < N_MIN` → `{skipped:true, flagged:[]}`); build
  profile (T1); `F = informativeFields`; per op `atypicality = 1 − Π_{f∈F present for op}(1 − a_f(op))`
  (noisy-OR); `flagged = ops with atypicality ≥ T_ABS`, sorted DESC, tie-break `rowIndex` ASC; per flagged op
  `reasons` = fields with `a_f ≥ REPORT_A_F`, ordered by ENT-021 strength (mcc→amount→text→counterparty/
  currency/bankCategory) then a_f desc, with the structured payload. Pure, SYNC, deterministic (no Date/random).
- **Tests (the headline proofs):**
  - **STOCK-MARKET-FEE:** a grocery bucket (mode MCC 5411, normal amounts/desc) + one op MCC 6051 → that op
    flagged, `reasons[0] = {field:'mcc', kind:'categorical-minority', value:6051}` — and it flags on MCC ALONE.
  - **HOMOGENEOUS → EMPTY tail** (all fields identical → all a_f 0 → empty).
  - **#6c HARDENED — homogeneous-category, VARIED descriptions → EMPTY tail** (same MCC/currency/normal
    amounts, but each op a different real-ish merchant string with IDs/numbers → text a_f high-but-CAPPED at
    0.5 < 0.6, all other a_f 0 → noisy-OR ≤ 0.5 → empty). This is the proof the text containment works.
  - N-gate (<8 → skipped); field-availability (null/constant → 0; filtered excluded; uniform-diverse no-flag);
    per-currency amount (a 10× UAH amount flagged; a USD op NOT cross-contaminated); determinism (byte-stable).

## Task 3 — `typicality/bucket.ts`: `bucketByWinningRule` (step 2)  [Wave A]
**Files:** `internal/rules/typicality/bucket.ts` (+spec).
- `bucketByWinningRule(rows, tree): Map<number, { rows: ImportStatementStage3Row[]; filteredFields: Set<TypicalityField> }>`
