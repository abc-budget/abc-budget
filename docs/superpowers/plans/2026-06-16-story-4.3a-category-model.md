## Task 4 — `CategoriesService` (CRUD + «base» + inline-create)  [Wave C — needs T3]
**Files:** `internal/categories/categories-service.ts` (+ spec). PORT prior-art `categories/service.ts`,
ADAPT (IoC → direct DI): constructor takes the `CategoriesDAO` + a base-currency resolver (the settings
`getBaseCurrencyOrNull(userSettingsDao)`); currency validation via `currency/reference.getCurrency(code)`.
- `create(category)`: validate (`name` non-blank → `InvalidCategoryError`; `currency` is a valid ISO via
  `getCurrency` **OR** the literal `'base'`; `isArchived` defaults false). **Generate `id =
  crypto.randomUUID()`** (classified user-action comment). DAO create. Store `currency` AS GIVEN (do NOT
  bake the base — the alias stays living).
- **`createFromSearch(name)` (ENT-018 inline create-from-search):** no match for `name` → create a new
  category (currency `'base'`, a default icon) + return it (auto-select). If a match exists, return it.
- `list({ includeArchived? })`: `getAll`, filter archived unless asked; **resolve `'base'`** in each
  returned category's `currency` (see below).
- `get(id)`: read + resolve `'base'`. `pick(id)`: alias for get (the selection accessor).
- **«base» resolve-at-read:** a private `resolveCurrency(cat)` → if `cat.currency === 'base'`, replace with
  `getBaseCurrencyOrNull(...)` (if null, leave `'base'` — base-not-set is an EP-1 setup concern; fail-soft,
  do not throw on read). Applied in `get`/`list`/`pick`, NOT in the stored row.
- `archive(id)` / `unarchive(id)`: read → flip `isArchived` → update. (Methods only; management UI = EP-7.)
- **Pins:** id is a `randomUUID` string; `icon` present, no image URL; a `'base'` category resolves to the
  base at read AND **follows a base change** (change base → re-read → new currency); inline
  create-from-search registers + auto-selects (and a second search for the same name returns the same, not
  a dupe); `list` filters archived; archive/unarchive flip the flag; `name` required throws.
