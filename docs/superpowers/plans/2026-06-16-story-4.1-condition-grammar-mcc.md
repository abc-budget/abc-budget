  `createIsBankCommissionRule`, `createIsCashbackRule`, and the `createStringMatchRule` /
  `createStringOperationRule` / `createBooleanOperationRule` helpers.
- **ADAPT:**
  - **+counterparty:** `createCounterpartyRule(op: StringOperation)`; widen `createStringOperationRule`

  - **−source:** do NOT port `createSourceRule` (Task 4 removes the row field).
  - **mcc categorical:** `createMccRule(op: StringMatchOperation)` = `createStringMatchRule('mcc', op)`

  - **amount↔currency (Fork B):** export `createAmountCondition(op: NumberOperation, currency: string):

  - **ReDoS (Fork C):** when a string factory gets `op.type === 'matches'`, call `assertSafeRegex(op.pattern)`

    `MAX_MATCH_INPUT`** (Task 3 constant).
- `index.ts`: internal barrel re-exporting operations + rule + factories. NOT added to the package
  public barrel (no rule types leak to the wire).
- **Tests:** every ENT-010 field × operator evaluates correctly (date×10, amount×7 via

## Task 3 — `safe-regex.ts` (ReDoS guard, Fork C)
**Files:** `internal/rules/safe-regex.ts` (+ spec).
- `export const MAX_MATCH_INPUT = 1000;` (bank fields are short; the **primary** mitigation — a cap so
  even an un-caught pattern can't run on a huge input).
- `export class UnsafeRegexError extends Error`.
- `export function assertSafeRegex(pattern: RegExp): void` — SECONDARY/best-effort: reject obvious
  catastrophic shapes on `pattern.source` — nested quantifiers `(x+)+` / `(x*)*` / `(x+)*` / `(x*)+`,
  a quantified group whose body contains a quantifier, overlapping alternation under a quantifier.
  Throw `UnsafeRegexError` on rejection.
- **Comment header MUST reframe the threat** per the ruling: user-authored regex over the user's OWN
  local data, single-user, worker-isolated → self-inflicted UX-robustness, not a security attack;
  the cap is primary, the heuristic is secondary and **explicitly not a complete ReDoS detector**.
- **Tests:** rejects `/(a+)+$/`, `/(a*)*$/`, `/(.*a){10}/`-style bombs; ACCEPTS ordinary patterns
  (`/coffee/i`, `/^ATB/`, `/\d{4}/`); `MAX_MATCH_INPUT` exported.
